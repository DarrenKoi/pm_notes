◆ 시험 안내

1. 시험 규칙

  • 시간: 90분
  • 개인전: 1인 1답안. 타 참가자와 협의 ✕
  • 사용 가능 도구:
  • ✅ 사내 RAG-LLM (Kimi-K2.5, Qwen3-VL 등 모델 선택 자유)
  • ✅ 일반 인터넷 검색
  • ✅ 본인 PC의 개인 메모·자료
  • ❌ 외부 LLM API (방화벽 차단)
  • ❌ 타 참가자와의 협의·자료 공유
  • 제출: 본 양식을 .md 파일로 사내 블로그에 업로드

2. 채점 기준 — 5축 각 20점 (총 100점)

축 평가 대상
1. Domain grounding 본인 팀의 구체적·검증 가능한 사례·수치가 박혀있는가
2. Framework application 5 stream / 암묵지 순환고리 / 3-tier 거버넌스 등 ITC 공통 프레임을 본인 맥락에 적용했는가
3. Critical refinement LLM의 generic 답을 본인 도메인 지식으로 교정·확장한 흔적이 본문에 직접 드러나는가
4. Meta-cognition 핵심 프롬프트 3개의 선택과 이유가 의식적 협업을 증명하는가
5. Narrative coherence 3개 영역이 하나의 thesis로 묶이는가 (단순 답변 나열 ✕)

3. ⚠ 본 시험의 핵심 강제사항

LLM이 제안한 generic 답을 그대로 옮기지 마십시오.

본인 도메인 지식으로 교정·확장·반박한 흔적이 본문에 명시적으로 드러나야 합니다.

권장 표현 예시: - "LLM은 X라고 제안했지만, 우리 팀 실제 경험상 Y에 더 가깝다 — 그 이유는…" - "처음에 ~로 물었을 때 답은 ~였으나, 우리 팀 맥락을 추가하니 ~로 바뀌었다"

이 흔적은 채점 축 3의 직접 평가 대상입니다. 본문에 0회면 축 3은 5점 이내로 한정됩니다.

4. ITC 공통 도메인 프레임 (채점 축 2의 기준)

다음 3개 프레임이 답안 전반에 정확히 적용되었는지를 평가합니다:

  • 5 stream: ① 의뢰 대응 ② 분석 ③ 데이터 생산 ④ 장비 운영 관리 ⑤ 장비 안정성 모니터링
  • 암묵지 순환고리 4단계: in-flow capture → 구조화·인덱싱 → RAG 활용 → 피드백
  • AI 확산 3-tier 거버넌스: 전사 AIX TF (DT 주도) / ITC AX Part / ITC AI 실행 TF

용어를 나열하는 것이 아니라, 본인 팀 사례에 적용하는 것이 평가 대상입니다.

◆ [영역 1] As-Is 진단 — 본인 팀 기준 (20분)

1-1. 본인 팀의 5 stream 비중과 가치 vs 시간 분석

  • 5개 stream 중 본인 팀에서 시간상 비중이 큰 stream은? 그 비중 추정치는?
  • 시간 비중과 후속 의사결정에 미치는 가치 비중이 다르다면, 어디서 차이가 나는가?

답:

1-2. 사내 RAG/LLM 인프라의 현재 활용도 — 본인 팀 관점

  • 현재 활용 가능한 도구(Kimi / Qwen-VL / BGE-M3 / 사내 RAG)별로, 본인 팀에서 실제로 막혀있는 지점은 무엇인가?
  • "활용 가능하다"가 아니라 "어디서 안 되는가"를 구체적으로.

답:

1-3. 🎯 [필수] 본인 팀이 최근 처리한 비정형 의뢰 1건 — 단건 detail

다음 4가지를 1건의 실제 사례로 작성하시오:

  1. 의뢰 내용 (1~2줄, 구체화 — 카테고리만 적지 말 것)
  2. 처리 과정의 단계와 각 단계 소요 시간
  3. 각 단계에서 기록된 산출물과 기록되지 않은 판단 근거
  4. 이 사례가 5 stream 중 어디에 해당하며, 왜 AIX 대상이 되는가
