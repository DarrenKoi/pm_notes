## [영역 1] 본인 Workflow + AIX 로드맵

### 1-1. 직무 Workflow 표

| 작업 유형 | 빈도 | AI 적용 가능성 | 적용 방식 | 5 stream |
|---|---|---|---|---|
| 신규 결함 패턴 근본원인 추정 | 월 ~12건 | **상** | 과거 *판단 근거* RAG 회수 → 시니어가 검증 | stream 2 (분석) |
| 정형 결함 분류 (기존 카테고리) | 월 ~300건 | 중 | Qwen3-VL 이미지 분류 + 사람 확인 | stream 2/3 |
| 계측 데이터 생산·정리 | 주 ~5회 | 하 | 자동화 여지 있으나 장비 종속 | stream 3 |
| 추가 계측 요청서 작성 | 월 ~20건 | 중 | 템플릿 + LLM 초안 | stream 1 |

### 1-2. AIX 적용 1순위

**1순위 = 신규 결함 패턴 근본원인 추정 (월 12건).** 빈도는 가장 낮지만, 한 건의 잘못된 추정이 양산총괄 후속 의사결정 며칠을 흔들어 **가치 비중이 압도적**. 정형 분류(월 300건)는 이미 도구화 진척이 있어 한계효용이 낮다.

> 기내식은 "월 300건 정형 분류가 빈도 높으니 1순위"라 답했으나, *빈도 ≠ 가치*. 본인 판단으로 저빈도·고가치 업무로 교정. (→ 축 3 흔적)

### 1-3. 로드맵 — 단기 / 장기

| 구분 | 내용 |
|---|---|
| **단기** | 근본원인 추정의 *판단 근거*를 슬롯 4개(`참조 패턴 ID / 유사 근거 / 상이 근거 / 가설 메커니즘`)로 in-flow capture. 분석 시간의 5% 미만으로. BGE-M3 인덱싱 → 신규 케이스에 top-3 회수. **진척 확인 = 캡처 건수 ≥ 100건 & RAG 회수율 ≥ 40%.** |
| **장기** | 회수된 판단 근거를 onboarding에 연결, 시니어→주니어 전수 시간 단축. (방향만) |

### 1-4. AI 적용 후 본인 업무의 방향

반복적 분류·1차 reference 검색은 AI로 넘기고, 본인은 **AI가 회수한 과거 판단을 *의심하고 교정*하는 검증자 + 새 가설을 세우는 사람**으로 이동. capture 슬롯 설계·품질 관리가 새 핵심 역량.

## [영역 2] 기내식 오답 검증 *(핵심)*

### 2-1. 기내식 답변 원문

> (인용) "DMI의 결함 분석 업무는 RAG에 과거 보고서를 색인하면 자동화됩니다. 과거 분석 결론을 검색해 유사 사례의 결론을 그대로 적용하면 TAT를 크게 줄일 수 있습니다."

### 2-2. 무엇이 왜 잘못/불완전한가

두 가지가 도메인적으로 틀렸다.

1. **"결론을 그대로 적용"이 위험.** 신규 결함은 *기존 카테고리에 안 맞아서* 신규다. 과거 *결론*을 그대로 적용하면 오분류 → 양산총괄 의사결정 오염. 우리 업무에서 회수해야 할 건 결론이 아니라 *판단 근거(왜 그렇게 봤는지)*다.
2. **"과거 보고서 색인"이면 충분하다는 전제가 틀림.** 현재 사내 RAG에는 *완성 보고서(명시지)*만 있고, 1-3의 슬롯형 *판단 근거(암묵지)*는 한 줄도 없다. LLM은 corpus에 무엇이 *없는지*를 모른다.

### 2-3. 본인은 어떻게 교정했는가

RAG의 출력 단위를 *결론*이 아니라 ***과거 시니어의 판단 근거 카드 3장*** 으로 재정의. 엔지니어가 그 근거를 *보고 직접 판단*하는 구조를 유지(human-in-the-loop). 그리고 색인 대상을 보고서가 아니라 in-flow capture 슬롯으로 바꿈 — 즉 "색인하면 끝"이 아니라 "*캡처 파이프라인을 먼저 만들어야* RAG가 의미 있다"로 순서를 뒤집음.

## [메타] 핵심 프롬프트 기록

### #1
**프롬프트**: `DMI 결함 분석 업무를 RAG로 자동화하는 가장 빠른 방법은?`
**영향**: 기내식이 "보고서 색인 → 결론 재사용"이라는 *틀린* 답을 줬고, 이게 영역2 검증의 재료가 됨. 일부러 generic하게 물어 LLM의 기본 가정을 노출시킨 게 결정적.

### #2
**프롬프트**: `우리 corpus에 '판단 근거'가 아니라 '결론'만 있다면, 그 RAG의 회수 결과는 신규 결함에 왜 위험한가?`
**영향**: 첫 답의 허점을 *내가 알고 있는 corpus 공백*으로 되물어, 결론 재사용의 위험을 LLM이 스스로 인정하게 만듦 → 2-2의 논리 구성.

### #3
**프롬프트**: `capture를 분석 시간의 5% 미만으로 묶으면서 4단계 판단을 다 담는 슬롯 설계 3개.`
**영향**: 제약을 박자 자유 서술 대신 슬롯형 4개라는 구체 설계가 나옴. 슬롯 명칭은 DMI 맥락으로 본인이 교정.

### LLM-only 샘플과의 차별점 (채점자 합의 포인트)

| 신호 | LLM-only | 모범 답안 |
|---|---|---|
| 영역2 인용 | 인용 없음 / "기내식이 정확했다" | LLM 답 원문 인용 + 도메인 2점 반박 |
| 수치 | "다양한", "지속적" | "월 12건", "5% 미만", "회수율 40%" |
| 로드맵 | "단기 도입 → 장기 확산" generic | 단기에 *측정 지표*까지 구체 |
| 프롬프트 | "X 정리해줘" | "오답 유도 → 공백 되묻기" 의도적 |

### 회색지대 (50~70점) 특징

- Workflow 표는 채웠으나 빈도가 비어있거나 "다수"
- 영역2에서 LLM 답을 인용은 했으나 반박이 일반론("좀 더 구체적이어야")에 그침
- 프롬프트 기록은 있으나 *영향*이 아니라 *내용 요약*만
