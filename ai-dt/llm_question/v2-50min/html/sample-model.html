1[영역 1] 반복업무 — Workflow + 로드맵

선택한 반복업무: 신규 결함 패턴 근본원인 추정

정형 결함 분류월 ~300건중Qwen3-VL 분류 + 사람 확인2/3

AIX 1순위: 신규 결함 근본원인 추정. 빈도는 낮지만 한 건의 오판이 후속 의사결정 며칠을 흔들어 가치 비중 압도적. 정형 분류(월 300건)는 도구화 진척이 있어 한계효용 낮음.

기내식은 "월 300건 정형 분류가 빈도 높으니 1순위"라 답했으나, 빈도 ≠ 가치. 본인 판단으로 저빈도·고가치 업무로 교정. (축3 흔적)

로드맵 — 단기: 근본원인 추정의 판단 근거를 슬롯 4개(참조 패턴ID / 유사 근거 / 상이 근거 / 가설)로 in-flow capture(분석 시간 5% 미만). BGE-M3 인덱싱 → 신규 케이스 top-3 회수. 진척 = 캡처 ≥ 100건 & 회수율 ≥ 40%.
장기: 회수된 판단 근거를 onboarding에 연결, 시니어→주니어 전수 시간 단축.

AI 적용 후 방향: 반복 분류·1차 검색은 AI로, 본인은 AI가 회수한 과거 판단을 의심·교정하는 검증자 + 새 가설 수립자로 이동. capture 슬롯 설계·품질 관리가 새 핵심 역량.

2[영역 2] 기내식 오답 검증 (핵심)

2-1. 기내식 답변 원문

"DMI의 결함 분석 업무는 RAG에 과거 보고서를 색인하면 자동화됩니다. 과거 분석 결론을 검색해 유사 사례의 결론을 그대로 적용하면 TAT를 크게 줄일 수 있습니다."

2-2. 무엇이 왜 잘못/불완전한가

  1. "결론을 그대로 적용"이 위험. 신규 결함은 기존 카테고리에 안 맞아서 신규다. 과거 결론 적용은 오분류 → 양산총괄 의사결정 오염. 회수해야 할 건 결론이 아니라 판단 근거다.
  2. "과거 보고서 색인이면 충분"이 틀림. 현재 RAG엔 완성 보고서(명시지)만 있고 슬롯형 판단 근거(암묵지)는 0. LLM은 corpus에 무엇이 없는지를 모른다.

2-3. 본인은 어떻게 교정했는가

RAG 출력 단위를 결론 → 과거 시니어의 판단 근거 카드 3장으로 재정의(human-in-the-loop 유지). 색인 대상도 보고서 → in-flow capture 슬롯으로 바꿔, "색인하면 끝"을 "캡처 파이프라인을 먼저 만들어야 RAG가 의미 있다"로 순서를 뒤집음.

3[메타] 핵심 프롬프트

#1 DMI 결함 분석을 RAG로 자동화하는 가장 빠른 방법은?
영향: 일부러 generic하게 물어 LLM의 틀린 기본 가정(보고서 색인→결론 재사용)을 노출 → 영역2 재료.

#2 corpus에 '판단 근거'가 아니라 '결론'만 있다면 RAG 회수가 신규 결함에 왜 위험한가?
영향: 내가 아는 corpus 공백으로 되물어 LLM이 위험을 스스로 인정 → 2-2 논리.

#3 capture를 분석 시간 5% 미만으로 묶으며 4단계 판단을 담는 슬롯 설계 3개.
영향: 제약을 박자 자유 서술 대신 슬롯형 4개 도출. 명칭은 DMI 맥락으로 교정.
