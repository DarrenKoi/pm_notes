## ◆ 시험 안내

### 1. 시험 규칙

- **시간**: **50분** (제출 여유 고려 — 실질 작성은 45분 안에 끝내는 것을 권장)
- **개인전**: 1인 1답안. 타 참가자와 협의 ✕
- **사용 가능 도구**:
  - ✅ 사내 RAG-LLM **"기내식"** (Kimi-K2.5, Qwen3-VL 등 모델 선택 자유)
  - ✅ 일반 인터넷 검색
  - ✅ 본인 PC의 개인 메모·자료
  - ❌ 외부 LLM API (방화벽 차단)
  - ❌ 타 참가자와의 협의·자료 공유
- **제출**: 본 양식을 .md 파일로 사내 블로그에 업로드

### 2. 이 시험이 측정하는 것 (1줄)

> **"기내식(LLM)이 만든 결과를, 본인 도메인 지식으로 검증·교정·확장할 수 있는가."**
>
> 화려한 AIX 기회를 가진 사람을 뽑는 시험이 **아닙니다**. 본인 업무를 정확히 분석하고, LLM의 답을 의심하고 고칠 줄 아는 사람을 평가합니다.

> 🟢 **시작 전 꼭 읽으세요 — 기내식을 *의심하면서* 작성하십시오.**
> 기내식의 답을 그대로 믿지 말고, 도메인적으로 어색하거나 틀린 답이 나오면 **그 자리에서 메모**해 두십시오. 그 답이 영역2의 재료가 됩니다. (45분이 끝난 뒤 "틀린 답이 뭐였더라" 떠올리려 하면 늦습니다.)
>
> 🟢 **기밀 표현**: 우수 답안은 센터 내 공유됩니다. 장비 절대 수치 등 민감 정보는 *범위·상대값*(예: "월 ~10건대", "TAT 절반")으로 표현해도 구체성 점수는 동일하게 인정됩니다.

### 3. 채점 기준 — *5축 비대칭, 총 100점*

| 축 | 배점 | 평가 대상 |
|---|---|---|
| **1. Domain grounding** | **25점** | 본인 업무의 *구체적·검증 가능한* 사례·수치가 박혀있는가 |
| **3. Critical refinement** | **25점** | LLM의 답을 본인 도메인 지식으로 *검증·교정한 흔적*이 드러나는가 (영역2 중심) |
| **4. Meta-cognition** | **20점** | 핵심 프롬프트 3개의 선택과 *영향* 설명이 의식적 협업을 증명하는가 |
| **2. Framework application** | **15점** | 5 stream / 암묵지 순환고리 / 3-tier 거버넌스를 *본인 맥락에 적용*했는가 (용어 나열 ✕) |
| **5. Narrative coherence** | **15점** | 영역1·2가 하나의 흐름으로 묶이는가 |

### 4. ⚠ 핵심 강제사항

> **(1)** LLM이 제안한 generic 답을 그대로 옮기지 마십시오. 본인 도메인 지식으로 *교정·확장·반박*한 흔적이 본문에 **명시적으로** 드러나야 합니다.
>
> **(2)** **「핵심 프롬프트 기록」 칸은 작성 중 실시간으로 채우십시오.** 빈칸이면 채점 축 4(20점)는 **0점** 처리됩니다. (시험 끝나고 재구성하기 어렵습니다.)
>
> **(3)** **AI 적용이 어려운 업무라면, *왜 어려운지를 도메인 근거로 정확히 분석*하는 것 자체가 만점 답안입니다.** 무리하게 AI를 갖다붙인 generic 로드맵은 오히려 감점됩니다.

### 5. ITC 공통 도메인 프레임 (채점 축 2의 기준)

본인 맥락에 *적용*하면 가점됩니다 (단순 나열은 가점 ✕):

- **5 stream**: ① 의뢰 대응 ② 분석 ③ 데이터 생산 ④ 장비 운영 관리 ⑤ 장비 안정성 모니터링
- **암묵지 순환고리 4단계**: in-flow capture → 구조화·인덱싱 → RAG 활용 → 피드백
- **AI 확산 3-tier 거버넌스**: 전사 AIX TF (DT 주도) / ITC AX Part / ITC AI 실행 TF
