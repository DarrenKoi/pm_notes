# LLM Report Contest

회사 내 LLM Chat 서비스인 기내식을 활용해 제한 시간 안에 리포트를 작성하는 경진대회 출제 컨텍스트.
핵심은 완성된 전략 문서 자체보다, 참가자가 LLM을 활용해 본인 업무 경험을 구조화하고 보고서로 완성하는 능력을 평가하는 것이다.

## Language

**LLM 리포트 경진대회**:
참가자가 제한 시간 안에 LLM을 활용해 본인 업무 기반 AIX 리포트를 작성하는 사내 대회.
_Avoid_: 일반 전략 보고서 공모전.

**기내식**:
회사 자체 구축 LLM Chat 서비스로, RAG를 통해 조직 업무 특성을 간략히 검색할 수 있는 도구.
_Avoid_: 외부 범용 LLM과 동일시.

**작성 시간**:
참가자가 실제로 문제를 읽고 LLM을 활용해 답안을 작성하는 시간 45분.
_Avoid_: 제출 포함 50분 전체를 작성 시간으로 계산.

**제출 시간**:
완성 답안을 저장하고 업로드하는 데 따로 둔 5분.
_Avoid_: 사고·작성·LLM 대화 시간에 포함.

**LLM 활용 능력**:
LLM을 사용해 리포트의 구조·근거·표현을 만들고, 본인 업무 맥락으로 구체화해 제출 가능한 답안으로 완성하는 능력.
_Avoid_: LLM 없이 혼자 쓴 도메인 지식 평가, 프롬프트 문구 암기 시험.

**핵심 프롬프트**:
참가자의 최종 리포트 방향·구조·근거를 바꾸거나 구체화하는 데 중요하게 작용한 프롬프트 2~3개.
_Avoid_: 보기 좋은 프롬프트 사후 나열, 전체 대화 로그 제출.

**LLM 활용 흔적**:
핵심 프롬프트마다 사용 이유, 반영한 LLM 답변 요약, 본인이 수정·보완·반박한 내용을 짧게 적은 메타 섹션.
_Avoid_: 전체 채팅 로그, 프롬프트만 단독 제출.

**3개 파트 답안 구조**:
45분 작성 시간에 맞춰 본문을 업무 Workflow 분해, AI 적용 지점과 적용 방식, 도입 후 역할 변화와 조직 관리 방안으로 압축한 답안 구조.
_Avoid_: 5개 주제를 모두 독립 대문항으로 나열.

**반복업무**:
참가자의 전체 직무 안에서 반복적으로 수행되어 Workflow 분해와 AI 적용 가능성 검토가 가능한 업무 1개.
_Avoid_: 직무 전체를 넓게 요약, 일회성 특수 사례만 선택.

## Relationships

- **LLM 리포트 경진대회**는 **작성 시간** 45분과 **제출 시간** 5분으로 운영된다.
- **기내식**은 참가자의 주요 LLM 도구이며, **LLM 활용 능력** 평가의 중심 도구다.
- **LLM 활용 능력**은 리포트 품질과 분리되지 않는다. 좋은 활용은 제출 가능한 좋은 리포트로 드러나야 한다.
- **핵심 프롬프트**는 최종 리포트의 어떤 부분을 바꿨는지 함께 설명되어야 **LLM 활용 능력**의 증거가 된다.
- **LLM 활용 흔적**은 **핵심 프롬프트** 2~3개를 중심으로 작성하되, 전체 LLM 답변 원문 제출은 요구하지 않는다.
- **3개 파트 답안 구조**는 기존 5개 관심사를 45분 안에 쓸 수 있게 압축한다.
- "AI 활용 능력 고과 반영 찬반"은 별도 대문항이 아니라 **3개 파트 답안 구조**의 조직 관리 방안 안에서 짧게 다룬다.
- 참가자는 본인 직무 전체를 넓게 다루지 않고, 그중 **반복업무** 1개를 골라 답안의 중심 사례로 삼는다.

## Example dialogue

> **출제자:** "50분 대회인데 보고서 자체를 잘 쓰는 사람을 뽑는 거야?"
> **Domain expert:** "아니요. 45분 안에 기내식을 잘 활용해서 본인 업무를 리포트로 구체화하는 능력을 보는 겁니다. 마지막 5분은 제출 시간입니다."
>
> **출제자:** "중요한 프롬프트 2~3개만 제출하게 하면 될까?"
> **Domain expert:** "좋습니다. 다만 그 프롬프트가 최종 리포트의 어떤 판단이나 문장을 바꿨는지까지 쓰게 해야 활용 능력 증거가 됩니다."

## Flagged ambiguities

- "50분"은 전체 운영 시간이지만, 실제 작성 타깃은 **작성 시간** 45분으로 분리됨.
- "좋은 리포트"와 **LLM 활용 능력**이 분리될 수 있음: 이 대회에서는 LLM 활용의 결과가 제출 리포트에 드러나야 한다.
- "중요한 프롬프트"는 문장이 좋아 보이는 프롬프트가 아니라, 최종 리포트에 영향을 준 **핵심 프롬프트**로 정의함.
- LLM 답변 원문 전체 제출은 45분 작성 시간을 갉아먹으므로 요구하지 않고, **LLM 활용 흔적**에 필요한 요약만 받는 것으로 해결함.
- 기존 `bgk.txt`의 5개 문제 후보는 그대로 독립 출제하지 않고, 45분 제약에 맞춰 **3개 파트 답안 구조**로 통합함.
- 업무 범위는 "본인 전체 직무"와 "아무 단건 업무" 사이에서, 본인 전체 직무 중 **반복업무** 1개를 선택하는 것으로 해결함.
