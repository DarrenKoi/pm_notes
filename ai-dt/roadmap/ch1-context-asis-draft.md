# Ch.1 배경 & As-Is 진단 — 슬라이드 드래프트 (v0.1)

> [Outline §Ch.1](./itc-aix-roadmap-outline.md#chapter-1-배경--as-is-진단--우리는-왜-변해야-하고-지금-어디-있나-3p) 본문 드래프트. **3슬라이드 분량**.

## Slide 1 — 세상은 이렇게 바뀌었다 (Slide 1.1)

### Slide title
**왜 지금 변해야 하는가 — 시장·기술 환경의 변화**

### Headline (1줄)
> **"반도체 미세화·소재 다양화·데이터 폭증과 생성 AI 실용화가 동시에 일어났다 — ITC가 마주한 3가지 압력."**

### 본문 좌측 (1/2) — 외부 변화 4개

| 영역 | 변화 |
|------|------|
| **공정** | 반도체 미세화 — 검사·계측의 정밀도·빈도 요구 ↑ |
| **소재** | 소재 다양화 — 실험·검증 cycle 폭증 |
| **데이터** | 검사·계측·FDC 데이터 폭증 (TB → PB 시대) |
| **AI** | 생성 AI·VLM의 실용화 — 추상적 기술에서 *바로 쓸 수 있는 도구*로 |

### 본문 우측 (1/2) — ITC가 마주한 3가지 압력

```
(a) Biz 의뢰량 증가  ↑↑↑
    vs
    인력 한계         →

(b) 계측·FDC 데이터 폭증  ↑↑↑
    vs
    활용도            →   (대부분 archive)

(c) 사내 AI 인프라 구축 완료  ✓
    →
    활용은 이제 시작
```

> **(c)가 가장 결정적**: GPU·LLM·플랫폼이 *이미* 준비되어 있다 — 우리는 *도구가 없어서 못 한* 게 아니라 *어떻게 쓸지를 결정하지 못한* 단계.

### Speaker notes
> "세 가지 압력 중 (a)와 (b)는 *위협*입니다 — 의뢰량은 늘어나는데 인력은 한정되고, 데이터는 쌓이는데 못 씁니다. (c)는 *기회*입니다 — 전사 AIX TF가 만든 인프라가 이미 가동 중. 이 압력의 합이 *지금 바로 움직여야 하는 이유* 입니다. 이 슬라이드의 메시지는 *'세상이 바뀌었으니 우리도 바뀌어야 한다'* 가 아니라 *'바뀔 도구가 이미 준비됐는데 우리가 이걸 어떻게 쓸지가 다음 1년의 문제'* 입니다."

### 슬라이드 우측 (시각요소 제안)
- 좌측: 외부 변화 4개 아이콘 + 짧은 라벨
- 우측: 3가지 압력 다이어그램 — (a)·(b)는 빨간 화살표 mismatch, (c)는 초록 ✓
- 하단 강조 박스: *"활용은 이제 시작"*

### [SLOT]
- 외부 변화 *"데이터 폭증"* 항목을 정량 anchor와 연결 (A2 데이터 월 생산량) — Slide 2의 anchor와 호응시킬지 결정

## Slide 2 — 우리는 지금 Passive다 ★★ (Slide 1.3 + 1.3a)

### Slide title
**우리는 지금 Passive다 — 5 Stream별 현재 한계와 Anchor 수치**

### Headline (1줄)
> **"양은 충분히 가지고 있으나, 활용 체계가 없다 — 5가지 흐름 모두에서 사람이 모든 일을 한다."**

### 본문 좌측 (2/3) — 5 Stream별 현재 한계

| Stream | 현재 한계 (Passive 상태) |
|--------|------------------------|
| 1. **의뢰 대응** | 의뢰 수기 분류·라우팅. 과거 사례는 사람 기억에 의존. TAT 편차 큼. |
| 2. **분석** | 보고서 처음부터 사람 작성. 과거 보고서 회수 비효율. |
| 3. **계측 데이터 생산** | 이미지·메트롤로지 데이터 대부분 *생성 후 archive* — 검색·재활용 체계 부재. |
| 4. **장비 운영 관리** | 운영 노하우가 *암묵지*로만 존재. 신규 인력 ramp-up 시간 길고 사고 위험. |
| 5. **장비 안정성 모니터링** | 알람 *사후 대응* — 분석되지 않고 종료되는 알람 다수. |

### 본문 우측 (1/3) — As-Is Anchor 3개 (수치 SLOT)

> **3-수치 전략 (Q17):** 정확한 anchor 3개 + 정성 진술 + 출처 tier.

```
┌─────────────────────────────────┐
│ A1 — 의뢰량 (Stream 1)          │
│ [SLOT: 연간 X건 → Y건]          │
│ [SLOT: 인당 Z건]                │
│ 출처: T1 (사내 ticketing)       │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ A2 — 계측 데이터량 (Stream 3)   │
│ [SLOT: 월 X TB → Y TB]          │
│ 최근 3년 추이 그래프            │
│ 출처: T1 (계측 시스템)          │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ A3 — FDC 사후분석률 (Stream 5)  │
│ [SLOT: X% 알람이 분석 미수행]   │
│ 출처: T2 (장비 운영 로그)       │
└─────────────────────────────────┘
```

**출처 tier:** T1 = 사내 시스템 raw / T2 = 자체 집계·과거 보고서 / T3 = 추정·인터뷰. CEO 보고에서는 **T1·T2만 anchor**, T3는 정성 보조.

### 본문 하단 박스 — 본질 메시지

> **"암묵지 미문서화가 본질적 병목이다."**
>
> 데이터·인프라는 있다. 부족한 것은 *엔지니어 머릿속의 노하우*를 RAG로 흐르게 하는 메커니즘. 1년차 Quick Win #5 — 암묵지 순환고리 PoC가 이 병목의 첫 검증이다.

### Speaker notes
> "이 슬라이드가 보고서 전체에서 가장 중요한 *진단* 슬라이드입니다. 좌측에서 5가지 흐름 모두가 *Passive*임을 보여드리고, 우측에서 그 진단을 *3개 수치로 정량화*합니다. 의뢰량은 [X]에서 [Y]로 늘었고, 계측 데이터는 [X]TB에서 [Y]TB로 늘었고, FDC 알람의 [X]%가 분석되지 않고 종료됩니다. *데이터가 없다*가 아니라 *데이터는 있는데 안 흐른다*가 본질입니다. 그리고 그 흐름의 가장 깊은 병목이 *암묵지*입니다."

### 슬라이드 시각요소 제안
- 좌측: 5 stream 아이콘 + 한 줄 한계 (작은 빨간 표시)
- 우측: anchor 3개 카드 (각 카드에 작은 그래프 — 3년 추이 라인)
- 하단 박스: *"암묵지 미문서화가 본질적 병목"* (시각적으로 분리, 다음 챕터로의 bridge)

### [SLOT]
- **A1·A2·A3 실값** — 사내 비공개 문서에서 주입. 드래프팅 단계에서는 SLOT 유지.
- **A1·A2·A3 그래프 형태** — 3년 추이 라인이 가장 강력. 막대보다 선호.
- **출처 표기 방식** — 슬라이드 footnote vs Appendix 단일 reference. 추천: footnote 짧게 + Appendix 상세.

## Slide 3 — 우리는 어디서 출발하나 (Slide 1.4)

### Slide title
**ITC의 AI 도입 준비도 — 5점 척도 진단**

### Headline (1줄)
> **"인프라·데이터는 강하다. 사람·문화·거버넌스가 다음 1년의 과제다."**

### 본문 좌측 (1/2) — 5점 척도 레이더 차트

```
              인프라
                ●  4.5
              ╱     ╲
       문화 ●         ● 데이터
        2.0           4.0
            ╲         ╱
              ●     ●
            거버넌스  인력
             2.5     3.0
```

### 5축 진단

| 축 | 점수 (5점 척도) | 근거 |
|----|----------------|------|
| **인프라** | [SLOT: 4.0~4.5] | 사내 LLM(Kimi/Qwen/BGE-M3)·GPU·플랫폼 구축 완료 |
| **데이터** | [SLOT: 3.5~4.0] | 5 stream 모두에서 데이터 풍부 — 단, 인벤토리·접근 표준 미흡 |
| **인력** | [SLOT: 2.5~3.0] | AI 챔피언 분산 보유 — 자체 개발 가능. 단, AI PM/PO 부재 |
| **문화** | [SLOT: 2.0~2.5] | 일부 팀 적극 / 일부 팀 관망. AIX/DX 격차 큼 (2차 동기) |
| **거버넌스** | [SLOT: 2.0~3.0] | 전사 AIX TF는 가동. ITC 내 AIX 위원회 미구성 |

### 본문 우측 (1/2) — 다음 1년의 과제

> **이 진단표가 그대로 [Enabler 5축](./itc-aix-roadmap-outline.md#chapter-5-enabler--거버넌스--조직-역량-강화-34p)의 우선순위가 된다.**

| 우선순위 | 축 | 1년 핵심 행동 |
|---------|----|--------------|
| **1** | 문화 | 전 직원 AI 리터러시 교육 + AI 실행 TF 정례화 (산하 6팀 10~20명) |
| **2** | 거버넌스 | AIX 추진 위원회(센터장 직속) 구성 + AI 사용 가이드라인 |
| **3** | 인력 | AI PM/PO 양성 시작 (AX Part 내) |
| **4** | 데이터 | 5 stream별 지식자산 인벤토리 + DRM 문서 VLM 추출 |
| **5** | 인프라 | (전사 TF 영역 활용, ITC 단독 행동 적음) |

### Speaker notes
> "마지막 슬라이드는 *희망적 진단*입니다. 인프라와 데이터는 이미 4점대 — 출발선이 좋습니다. 약점은 사람·문화·거버넌스. 다행히 이 셋은 *돈으로 못 사지만 1년 안에 만들 수 있는 것* 들입니다. 다음 챕터에서 보여드릴 로드맵은 이 진단표를 그대로 *Enabler 5축*으로 옮겨 옵니다."

### 슬라이드 시각요소 제안
- 좌측: 레이더 차트 (5축, 현재 점수)
- 우측: 1년 우선순위 표 (1·2·3 강조, 4·5는 약하게)
- 색상: 4점 이상은 초록, 3점대는 노랑, 2점대는 빨강

### [SLOT]
- **5축 점수 실값** — 자체 진단 또는 외부 컨설팅 결과로 결정. 드래프팅 단계에서는 범위만 표시.
- **레이더 차트 vs 막대 차트** — 레이더가 *"균형/불균형"* 시각적으로 강함. 권장.
