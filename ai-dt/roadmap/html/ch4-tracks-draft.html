SLIDE 1 Overview Matrix

3개 트랙으로 5가지 능력을 횡전개한다

“같은 5 능력(분류·RAG·초안·이상·추천)을 3개 영역에 단계적으로 배치 — 공통 능력 한 번 만들어 횡전개.”

5 능력 × 3 트랙 매트릭스 — 파랑 = Active (1~2년), 보라 = Proactive (3~5년)

트랙 무게

Track 1 — 의뢰1.5p
★ Track 2 — 장비·공정3p
★ Track 3 — R&D 가속3p
Track 1 (1.5p) — 전 stream 공통 entry point. 모든 팀이 의뢰 대응을 하므로 시작점이 단순·균일.
Track 2 ★ (3p) — 공정·장비 안정성 직결. 수율·downtime impact 가장 큼. Hero KPI 3개 중 2개(★3·5)가 여기.
Track 3 ★ (3p) — R&D 가속. Proactive 능력(추천·시뮬레이션)의 주 무대. 3~5년차 ROI 가장 큼.
Speaker notes

"이 매트릭스가 추진 트랙 챕터의 전체 청사진입니다. 가로축은 5 능력, 세로축은 3 트랙. 같은 능력이 여러 트랙에 등장한다는 점이 핵심 — 예를 들어 RAG는 3 트랙 모두에 들어갑니다. 이게 의미하는 바는 공통 능력 한 번 만들어 세 곳에 옮긴다입니다. 트랙별로 별도 RAG를 만드는 게 아닙니다."

SLIDE 2 Track 1 — Biz 의뢰 능동 대응 (1.5p → 1 슬라이드)

Track 1 — Biz 의뢰 능동 대응

“의뢰가 들어오면 자동 분류·라우팅, 과거 사례 회수, 응답 초안화 — 1~2년차 핵심 entry point.”

Active 능력 1~2년

분류·선별
  • 의뢰 자동 라우팅 — 의뢰 유형 N개 다중 분류
RAG
  • 과거 사례·해석 의미 기반 회수
초안 생성
  • 응답·해석 보고서 초안화

Proactive 능력 3~5년

이상·예측
  • Biz KPI 모니터링 → "이 패턴은 사전 대응 필요" 알림
추천·시뮬레이션 — (현재 적용 없음)

KPI — Track 1

KPI1년3년
의뢰 평균 TAT Hero 1↓30%↓50%
분류 정확도≥85%≥95%
자동 라우팅 비율≥50%≥80%
"Track 1은 모든 팀의 entry point"
AIX TF 공통 능력이 가장 빠르게 흐를 수 있는 통로. Quick Win #1과 직접 연결.
Speaker notes

"Track 1이 1.5p로 가장 짧은 이유는 모든 팀이 의뢰를 받기 때문에 entry point로서의 균일성·단순성이 높기 때문입니다. 6팀 모두에 같은 패턴이 적용 가능합니다. 1년차 Quick Win #1이 여기에 위치합니다."

SLIDE 3 ★ Track 2 — 장비·공정 AI 운영 (Active, 3p 중 1/2)

Track 2 — 장비·공정 AI 운영 Active

“검측 이미지·알람을 자동 분류, 운영 매뉴얼을 RAG로 응답, 검측 리포트를 초안화한다.”
분류·선별
QW #3
  • 검측 이미지 다중 분류 (정확도 ≥90%)
  • 알람 자동 카테고라이즈 + 우선순위 큐
데이터 의존성: A2 (계측 데이터 인벤토리)
RAG
  • 운영 매뉴얼·과거 셋업 로그·교대 인계 노트 의미 기반 회수
  • 신규 인력 ramp-up 시간 단축 (L1 KPI Stream 4)
데이터 의존성: 운영 매뉴얼 인벤토리 (Enabler 5.3a 명시지)
초안 생성
QW #2
  • 검측 결과 → 자동 리포트 초안 → 분석가 편집
  • 분석 보고서 작성시간 ↓40% (L1 KPI Stream 2)
"Track 2 Active = 검측·분석·운영 3개 stream을 동시에 가속하는 1~2년차 핵심 트랙."
Speaker notes

"Track 2가 가장 무거운 트랙입니다. 이유는 단순합니다 — 5 stream 중 3개(분석·계측·운영)가 여기 모두 들어 있기 때문. 1년차 Quick Win 3개가 이 슬라이드 안에 있습니다. Active 단계에서 분류·RAG·초안 3개 능력을 모두 가동합니다."

SLIDE 4 ★ Track 2 — 장비·공정 AI 운영 (Proactive, 3p 중 2/2)

Track 2 — 장비·공정 AI 운영 Proactive

“FDC 시계열을 학습해 *고장 전에* 알림 — 예지보전과 OPC 보조.”
이상·예측 — FDC 기반 예지보전·결함 예측
QW #4 시드
  • FDC 시계열 이상 탐지 모델 — 알람 전 시그널 캡처
  • 결함 예측 — 특정 조건에서 결함 발생 확률
  • 1년차 시드 PoC "완성 아님, 시그널 검증"
  • 3년차 본 적용: 예지보전 hit rate ≥40% (★ Hero 5)
데이터 의존성: A3 (FDC 사후분석률) + 한정 범위 장비 라벨
추천·시뮬레이션 — OPC 보조·검사 우선순위
  • OPC 시뮬레이션 보조 — 특정 조건 검색·추천
  • 검사 우선순위 자동 산출 — 위험도 기반 큐 정렬
  • 1~2년차에 baseline 측정, 3~5년차에 본 적용

KPI — Track 2 Proactive

KPI1년3년
예지보전 hit rate Hero 5≥40% (시그널 검증)≥70%
알람 평균 대응시간 Hero 5↓25%↓50%
검측 자동 분류 정확도≥90%≥95%
Speaker notes

"Proactive 단계의 Track 2는 고장 전에 알림이라는 한 줄로 표현됩니다. 이게 downtime 절감의 직접 경로이고, Hero KPI 2개가 여기에 있습니다. 1년차에는 #4 시드 수준 PoC만 — 3년차에 본 적용 조건이 발견됩니다."

SLIDE 5 ★ Track 3 — R&D 가속 (Active, 3p 중 1/2)

Track 3 — R&D 가속 Active

“사내 문헌·실험 사례를 RAG로 회수, 분석 리포트를 초안화, 트렌드를 자동 탐지한다.”
RAG — 사내 문헌·실험 사례 회수
  • 사내 R&D 보고서·실험 노트·외부 문헌 의미 기반 회수
  • 실험 cycle 단축의 직접 도구
데이터 의존성: ITC 통합 지식 그래프 (Enabler 5.3a) 부분 가동
초안 생성 — 분석 리포트
  • R&D 분석 결과 → 자동 리포트 초안
  • Track 2 Slide 3의 검측 리포트 초안과 동일 능력 횡전개
데이터 의존성: 과거 R&D 보고서 N년치
분류·선별 — 트렌드 자동 탐지
  • 실험 결과·외부 publication에서 의미 있는 트렌드 자동 탐지
  • "이 영역에서 최근 X% 증가" 같은 시그널 자동 산출
"Track 3 Active = R&D 엔지니어가 문헌·과거 사례 검색에 쓰던 시간을 실험·해석으로 옮긴다."
Speaker notes

"Track 3는 R&D 엔지니어의 일하는 시간 분포를 바꾸는 것이 목표입니다. 현재는 검색·정리에 많은 시간이 소요됩니다. RAG와 초안 생성이 그 시간을 압축해주면, 그 시간은 진짜 실험과 해석으로 흐릅니다."

SLIDE 6 ★ Track 3 — R&D 가속 (Proactive, 3p 중 2/2)

Track 3 — R&D 가속 Proactive

“AI가 조성·공정 조건 후보를 *먼저* 제안 — R&D cycle이 제안 → 검증으로 단축된다.”
추천·시뮬레이션 — 조성·공정 조건 후보, 실험 우선순위
  • AI가 "이 소재 조성이 우선순위 1위" 같은 추천 산출
  • 실험 우선순위 자동 정렬 — 가능성 × 가치 × 비용
  • 시뮬레이션 보조 — 일부 실험을 digital twin으로 대체
  • 3~5년차 본 적용. 1~2년차에는 baseline·평가 체계 구축
이상·예측 — 신규 결함 가능성
  • 새 소재·새 공정 조건에서 결함 발생 가능성 예측
  • Track 2 Proactive와 동일 능력 재사용 — 학습 데이터만 다름

KPI — Track 3 Proactive

KPI1년3년
실험 cycle 단축률baseline 확보↓30%
문헌 회수 정확도≥80%≥95%
후보 추천 채택률baseline≥30%
Speaker notes

"Proactive Track 3는 AI가 R&D의 일부가 되는 단계입니다. 추천된 후보를 엔지니어가 검증·채택합니다. 채택률이 KPI인 이유는 추천이 좋아야 사람이 받아들이기 때문 — 단순 정확도가 아닌 수용성이 측정됩니다."
