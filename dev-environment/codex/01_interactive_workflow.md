# 대화형 작업 흐름 (Interactive Workflow)

> Codex를 가장 잘 쓰는 방법은 "질문"이 아니라 "작업"을 맡기는 것이다.

## 왜 중요한가? (Why)

- 대화형 모드는 코드베이스 탐색, 계획 수립, 수정, 검증을 여러 턴에 걸쳐 이어갈 수 있다
- 초반에 범위와 제약을 잘 잡아주면 불필요한 파일 변경과 장황한 응답을 줄일 수 있다

## 시작 패턴 (How)

### 가장 기본

```bash
codex
```

현재 디렉토리를 작업 루트로 잡고 대화형 세션을 시작한다.

### 프롬프트를 바로 넣고 시작

```bash
codex "이 저장소 구조를 먼저 파악한 뒤, 배포 관련 설정 파일만 찾아서 설명해줘."
```

처음부터 해야 할 일을 분명히 주면 첫 턴의 탐색 품질이 좋아진다.

### 작업 디렉토리를 명시

```bash
codex -C web-development/python/flask/job-scheduler
```

가장 중요한 옵션 중 하나다. 저장소가 크면 반드시 `-C`로 범위를 줄이는 편이 좋다.

### 추가 쓰기 가능 디렉토리 부여

```bash
codex -C web-development/python/flask/job-scheduler \
  --add-dir /tmp
```

기본 작업 루트 외에 추가 출력 경로가 필요할 때 사용한다.

### 스크롤백 보존

```bash
codex --no-alt-screen
```

tmux, zellij, 긴 세션 기록 확인이 중요할 때 유용하다.

## 잘 먹히는 첫 요청 템플릿

Codex는 아래 5요소가 들어간 요청에서 특히 안정적으로 동작한다.

1. 목표: 무엇을 바꿀지
2. 범위: 어느 폴더/기능만 볼지
3. 제약: 건드리면 안 되는 부분, 선호 기술, 스타일 규칙
4. 검증: 어떤 명령으로 확인할지
5. 완료 조건: 최종 응답에 무엇이 포함되어야 하는지

예시:

```text
job-scheduler 백엔드에서 실패한 스케줄 재시도 로직만 수정해줘.
frontend는 건드리지 말고, 기존 스타일을 유지해.
먼저 관련 파일을 찾아 원인을 설명한 뒤 수정하고,
가능하면 테스트나 실행 명령으로 검증해줘.
마지막에는 변경 이유와 검증 결과만 짧게 정리해줘.
```

## 효과적인 운영 패턴

### 1. 첫 턴에서는 "바로 수정"보다 "먼저 파악"

좋은 예:

```text
먼저 관련 파일과 현재 동작을 파악하고, 수정 계획을 짧게 제시한 뒤 작업해줘.
```

큰 저장소나 낯선 코드에서는 이 한 문장으로 시행착오를 많이 줄일 수 있다.

### 2. 범위를 파일/모듈 단위로 좁힌다

좋은 예:

```text
`frontend/`는 제외하고 `src/`와 `tests/`만 봐줘.
```

나쁜 예:

```text
전체 프로젝트를 보고 문제를 다 고쳐줘.
```

### 3. "설명"보다 "검증 가능한 결과"를 요구한다

좋은 예:

```text
수정 후 실행한 명령, 실패/성공 여부, 남은 리스크를 알려줘.
```

### 4. 출력 형식을 짧게 고정한다

```text
최종 답변은 변경 사항, 검증 결과, 남은 이슈 3가지만 포함해줘.
```

이렇게 하면 장황한 설명이 줄고 회고하기 쉬워진다.

## 자주 쓰는 실전 예시

### 버그 수정

```bash
codex -C web-development/python/flask/job-scheduler \
  "백엔드에서 다음 실행 시간이 잘못 계산되는 버그를 찾아 수정해줘.
  원인 설명 후 수정하고, 가능한 검증 명령까지 실행해줘."
```

### 코드베이스 이해

```bash
codex -C Codes/python/opensearch_handler \
  "이 모듈의 엔트리포인트, 주요 클래스, 외부 의존성을 정리해줘.
  README 갱신이 필요하면 같이 수정해줘."
```

### 문서 개선

```bash
codex -C terminal \
  "문서 목차와 링크가 끊긴 부분이 있는지 확인하고 README를 개선해줘."
```

## 피해야 할 패턴

- 목표 없이 "이 코드 설명해줘"만 던지기
- 작업 폴더를 좁히지 않고 저장소 전체를 대상으로 시작하기
- 승인 정책을 이해하지 않은 채 위험한 자동 실행 모드부터 켜기
- 검증 요구 없이 수정만 시키기

## 한 줄 요약

대화형 모드의 핵심은 `정확한 작업 루트 + 명확한 완료 조건 + 검증 요청`이다.
