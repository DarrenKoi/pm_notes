# Roo Code 작업 메모 템플릿

DRM 원문 문서를 직접 넣지 않고, 사용자가 공유 가능한 범위에서 핵심 내용을 정리한 뒤 Roo Code와 작업하기 위한 템플릿이다.

## 1. 작업 목적

- 만들고 싶은 산출물:
- 사용할 자리:
- 발표/문서 대상자:
- 시간 또는 분량:

## 2. 배경

- 왜 이 작업이 필요한가:
- 현재 논의되고 있는 문제:
- 이미 결정된 사항:
- 아직 결정되지 않은 사항:

## 3. 공유 가능한 핵심 내용

- 핵심 사실 1:
- 핵심 사실 2:
- 핵심 사실 3:

## 4. 반드시 지켜야 할 제약

- 보안상 언급하면 안 되는 내용:
- 회사 정책상 조심해야 할 표현:
- 외부 도구나 외부 API 사용 가능 여부:
- 모델이 직접 열람하면 안 되는 자료:

## 5. 원하는 출력 형식

예시:

```text
10분 발표용 슬라이드 10장 구성으로 작성해줘.
각 슬라이드는 다음 형식으로 작성해줘.

## 슬라이드 번호. 제목
- 슬라이드 메시지:
- 본문 bullet:
- 발표자 노트:
```

## 6. Roo Code에 요청할 프롬프트

```text
아래 작업 메모를 기준으로 발표 초안을 만들어줘.
대상자는 기획, 운영, 관리자이고 발표 시간은 10분이야.
보안상 위험한 표현은 피하고, Agent와 Harness 개념을 쉽게 설명해줘.
Roo Code는 사내 LLM API를 연결해 Agent형 작업 방식을 익히는 예시로 소개해줘.
```

## 7. 후속 작업

- Roo Code에서 초안 생성
- `/grill` 또는 `/grill-with-docs`로 논리 점검
- Markdown 초안을 PPT Agent에 전달
- 사람이 보안/사실관계/표현 최종 검토
