# 사내 LLM으로 시작하는 AI Agent 활용

10분 팀 미팅 발표용 자료 초안이다. 발표 대상은 기획, 운영, 관리자이며, Roo Code를 개발 IDE 심화 도구가 아니라 사내 LLM API로 Agent형 작업 방식을 익히는 현실적인 출발점으로 소개한다.

## 핵심 메시지

외부 상용 Agent 도구를 그대로 쓰기는 어렵지만, 사내 LLM API와 Roo Code 같은 Agent Harness를 연결하면 개인 PC에서도 보안 경계 안에서 Agent형 작업 방식을 실험할 수 있다.

중요한 것은 특정 도구 하나가 아니라 다음 습관이다.

- 모델과 대화하며 업무 목적을 구체화한다.
- 업무 컨텍스트를 재사용 가능한 텍스트로 쌓는다.
- 산출물을 한 번에 완성하려 하지 않고 반복적으로 발전시킨다.
- 필요한 경우 PPT Agent 같은 후속 도구로 넘길 수 있는 초안을 만든다.

## 발표 운영 원칙

- 본편은 10분, 10장, 메시지형 슬라이드로 진행한다.
- 설치와 상세 설정은 본편에서 설명하지 않고 부록으로 남긴다.
- Claude Code, Codex, Gemini CLI는 비교 대상이 아니라 Agent Harness 흐름의 사례로만 언급한다.
- DRM 원문 파일을 Agent가 직접 열람한다고 말하지 않는다.
- 사용자가 공유 가능한 범위의 핵심 내용을 텍스트 작업 메모로 정리한 뒤 Roo Code와 협업하는 흐름을 강조한다.
