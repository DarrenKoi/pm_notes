### 상세
- **Align Fail 자동화 현업 투입**
  - 내용: 상반기 GUI 자동화 완성에 이어, align 이미지 기반 위치 재조정 정확도를 끌어올려 PoC를 완성하고 현장에 무인 처리로 정착.
  - 목표: 주 ~400건 무인 처리 안정화, 실패 케이스는 Cube 에스컬레이션으로 흡수.
  - 일정: 6~7월 PoC 완성 → 하반기 현업 투입.

- **SKEWNONO 완성·안정화**
  - 내용: 7월초 재구축 완료(서버 이관 + 신규 기능) 후 운영 전환, MI 현업 요구사항 추가 반영.
  - 목표: 안정 운영 + CD-SEM 데이터 활용도 향상.
  - 일정: 7월초 완성 → 하반기 운영·개선.

- **OSS-Cube 고도화**
  - 내용: 현재 개발 용이성을 위한 개발 환경 고도화에 집중, 이어서 웹에서도 사용 가능하도록 확장. 장기적으로 Cube AI Agent 연동.
  - 목표: 자체 솔루션들이 쉽게 올라타고 웹·Cube 양쪽에서 활용되는 공통 인프라 완성.
  - 일정: 하반기.

## 2. 하고 싶은데 못하고 있는 업무 (사유 포함)

### 2-1. 기획 업무 학습·경험
- **하고 싶은 이유 / 기대 효과**: 개발 실무를 넘어 기획 역량까지 확장하면 AI/DT 확산을 더 큰 그림으로 주도 가능. 커리어 성장.
- **못하고 있는 사유**: 개발 실무에 집중하느라 기획 참여 기회를 놓침. 게다가 **실질적으로 개발 가능한 인력이 적어, 기획을 하더라도 결국 본인이 개발까지 떠안는 구조**.
- **하기 위해 필요한 것**: 개발 인력 확보(기획에 집중할 여유 확보) + 기획 업무 참여 기회 배정.

### 2-2. 현업 엔지니어 개발 요청 신속 처리
- **하고 싶은 이유 / 기대 효과**: 요청을 빠르게 처리할수록 AI/DT 확산의 체감·신뢰·솔루션 채택률이 높아짐.
- **못하고 있는 사유**: 혼자 개발+운영을 병행해 요청량 대비 처리 여력 부족(상반기 Lowlight와 동일 뿌리).
- **하기 위해 필요한 것**: 개발/운영 백업 인력, 요청 접수·우선순위(트리아지) 체계.

## 3. 팀장에게 하고 싶은 얘기

- **개발/운영 백업 인력 충원** — 서비스가 늘수록 유지보수에 시간이 쏠려 신규 개발과 현업 요청 대응이 잠식되고 있음. 안정화된 서비스 운영을 분담할 인력이 있어야 AI/DT 확산이 지속 가능.
- **기획 경험 기회 부여 (전제: 개발 인력 확보)** — 기획 역량으로 확장하고 싶으나, 실질 개발 인력이 적어 기획해도 본인이 개발까지 해야 하는 현실. 개발 인력이 확보돼야 기획에 집중·학습할 수 있음. → 인력 충원 한 건이 운영 부채와 역량 성장 정체를 동시에 해결.
- **현업 요청 창구·우선순위 체계화** — 요청이 개인에게 직접 몰리는 구조를 팀 차원의 접수/우선순위 체계로 정리하면 처리 속도와 형평성이 개선됨.
