# 원문 05 — [참고] 프로세스 레벨 정의 (4/9)

> 예, 영업 분야 Process Level 정의 예시. L1~L5의 5단계 위계로 업무를 분해.

| L1 (Mega Process) | L2 (Process Group) | L3 (Process) | L4 (Activity) | L5 (Task) |
|-------------------|--------------------|--------------|---------------|-----------|
| 영업 | 시장 및 고객 개발 | 시장 분석 및 목표 고객 설정 | 고객 세그먼트 분석 | 내부 CRM Data 추출 및 정제<br>외부 시장 Data 수집<br>매력도 접근성 분석 |
| | 영업 기회 관리 | 리드 발굴 및 육성<br>고객 니즈 파악 및 기회 검증 | 고객 미팅 및 니즈 인터뷰 | 미팅 전 고객 사전 조사<br>니즈 인터뷰 실시<br>미팅 결과 CRM 입력 |
| | | 제안서 및 견적 작성 | 견적 산출 및 가격 승인 | 표준가격기반 견적초안 작성<br>특별할인 승인 요청<br>최종 견적서 완성 및 발송 |
| | 납품 및 고객 서비스 | SO 생성 및 확인<br>납품 및 납기 관리 | 출하 지시 및 물류 조율 | ERP 출하지시서 생성<br>3PL 운송 예약 및 확인<br>조건 대응 및 최종합의 문서화 |
| | 수금 및 채권 관리 | 클레임 및 VOC 처리<br>청구서 발행 및 대금 청구 | 세금 계산서 발행 | 납품 완료 및 청구 기준 검토<br>ERP 청구 문서 생성<br>전자세금계산서 발행 및 발송 |
| | | 채권 회수 및 연체 관리 | | |

**해석 포인트**: AI Agent의 "개입 최소 단위"는 L5(Task) 수준. Task까지 분해해야 어디에 AI를 배치할지 정밀하게 식별 가능.
