# 09 · KSF·제약 정의 — 발표 장표 (2매)

> [09-ksf-제약-정의.md](./09-ksf-제약-정의.md) 산출물을 발표용 2장으로 압축한다. 템플릿 2산출물(① 4관점 KSF 매트릭스 · ② 선결 우선순위 #1~#4)을 **장표 1 = ①**, **장표 2 = ② + 안전·보안·감사 보강** 으로 나눈다. 적용 기술은 사내 LLM만(Qwen3-VL·BGE-M3). 수치(τ/r/s/N)는 캠프·PoC 확정 placeholder. `## 장표 N` = 슬라이드 1장.

---

## 장표 1 — KSF 매트릭스 (4관점 × 핵심성공요소·장애요인·선결조건·적용기술)

To-Be(#08)가 작동하려면 4관점(**업무 프로세스·데이터·시스템/인프라·조직 문화**)에서 무엇이 선결돼야 하는가를 박제한다 — **반드시 충족돼야 작동하는 must** 위주(있으면 좋은 것 아님). 장애 요인 열은 #10 리스크 게이트(🚦) 입력.

| 관점 | 핵심 성공요소 (KSF) | 장애 요인 | 선결 조건 | 적용 기술 |
|---|---|---|---|---|
| **업무 프로세스** | ②~⑥ 연속 수행 + 예외 시 기준대로 사람 핸드오프 | ⑥ 좌표 재정합 잔존 오류 · 예외 기준 모호 | 신뢰도 임계 τ·재시도 r 정의 + ⑦ 승인 게이트 확정 | 신뢰도 룰엔진 · 핸드오프 오케스트레이션 |
| **데이터** | SEM 이미지·ref recipe·라벨 충분 확보 | 배율·노이즈 편차 · 라벨 공수 · 사내문서 99% **DRM** | 숙련 엔지니어 라벨 + 스크린샷→VLM 추출 파이프라인 | Qwen3-VL 추출 · BGE-M3 검색 |
| **시스템·인프라** | GUI 자동제어 안정 + VLM 추론 자원 확보 | 장비 SW 버전 의존 · GPU 한계 · **외부 LLM API 차단** | 사내 Qwen3-VL 엔드포인트 · GPU 서버 · 섀도우 환경 · 안전 인터록 | Qwen3-VL-8B/30B · computer-use |
| **조직 문화** | 엔지니어 신뢰 + 현장 수용 | 장비 직접 제어 거부감 · 검증 권한 귀속 | ⑦ **사람 최종승인 유지** · 운영팀 협업 · 단계적 도입 | 설명가능 리포트 · HITL 승인 UI |

> **정합성 박제** — 조직 문화 선결 「⑦ 사람 최종승인 유지」 = #08 To-Be 🧑 레인과 **동일**(불일치 = 설계 오류) / 장애 요인 열 = #10 리스크 게이트(🚦) 입력 / 적용 기술 LLM은 전부 **사내 OpenAI-호환 엔드포인트**(외부 API 없음).

## 장표 2 — 선결 우선순위 (#1~#4) + 안전·보안·감사 보강

무엇을 **먼저** 검증·확보할지 순서. **#1은 다른 모든 선결의 전제** — 안 되면 #2~#4를 채워도 ⑥ 자동화가 성립하지 않는다.

| 순위 | 선결 항목 | 성격 · 연결 |
|:---:|---|---|
| **#1** | 좌표 재정합 정확도 (τ) | 최대 리스크 · ⑥ 자동화 성립 여부 자체 = **Phase0 PoC 검증**(04 S12·05), 실 장비 영향 0 섀도우, 미달 시 재설계 게이트 |
| **#2** | GUI 자동제어(computer-use) 안정 | ②~⑤ 무중단 · 장비 SW 버전 의존성 관리 |
| **#3** | 핸드오프 기준 (예외 임계 τ/r) | "어느 오인식부터 사람에게 넘기나" 정의 |
| **#4** | 신뢰·수용 (현장 도입) | 엔지니어 신뢰 + 운영팀 협업 + 단계적 도입 |

**■ 안전·보안·감사 보강** (VLM이 장비를 직접 제어하는 과제이므로 필수)

| 항목 | 선결 내용 |
|---|---|
| **안전 인터록·롤백** | 실 웨이퍼·라인 영향 동작은 인터록 사전 차단 + 오제어 즉시 롤백 (섀도우/오프라인 우선 검증) |
| **사고 책임소재** | AI 오작동 시 책임자·대응 절차 사전 지정 (`<담당 임원>`·운영팀, #10 RACI 연결) |
| **보안 승인·감사 추적** | DRM→VLM 경로 유출 점검 + 사내 엔드포인트 격리·접근로그 **보안팀 승인**, AI 판정은 모델 버전과 함께 감사 로그로 재현 가능 |

> τ(재정합 정확도)·r(재시도)·s/s′(소요시간)·N(처리량)은 캠프/PoC에서 엔지니어 확정 placeholder — 본 장표는 변수와 검증 순서만 고정한다.
