# 11 · Review — 발표 장표 (2매)

> [11-review-산출물-발표.md](./11-review-산출물-발표.md) 산출물을 발표용 2장으로 압축한다. 템플릿 2부(A 설계 검증 체크리스트 6질문 · B 마무리)를 **장표 1 = A 체크리스트**, **장표 2 = 교차 정합성 + B 마무리(피드백 루프)** 로 나눈다. Review는 신규 생성이 아니라 **검산** — ✅ 미충족 = 출처 파일 부실 신호. `## 장표 N` = 슬라이드 1장.

---

## 장표 1 — 설계 검증 체크리스트 (6질문 × 판정 · 근거)

원문 10의 3대 점검(역할 구분·병목 개선·제약 다각도)에 본 과제 고유 3점검(KSF 명확·전략 추적성·가치 정량)을 더한 6질문. **질문은 고정, 판정·근거만 채운다.** ✅ = 출처 파일이 답함 / △ = 구조는 잡혔으나 값 placeholder / ❌ = 회귀 보강.

| # | 점검 질문 | 판정 | 근거 — 어느 산출물이 답하나 |
|:---:|---|:---:|---|
| 1 | AI·사람 역할이 Task별로 명확히 구분되는가? | ✅ | #08 🤖/🧑 Swimlane(②~⑥=AI / ①·⑦·예외=사람) + #02 AI 대체 판정 |
| 2 | 최대 병목 구간을 개선하는가? | ✅ | #03 ⑥ 최대 병목 식별 → #07 s₆=최대 → #08 ⑥ VLM 실시간 재정합 대체 |
| 3 | 제약을 다각도로 검토하는가? | ✅ | #09 4관점(업무 프로세스·데이터·시스템/인프라·조직 문화) 장애·선결 |
| 4 | KSF가 선결 조건 위주로 명확한가? | ✅ | #09 선결 우선순위 #1~#4 + #05 준비계획 구체성(PoC) |
| 5 | 과제가 전략에서 연역됐는가(추적성)? | ✅ | #01 Top-Down 연계표(출발) ↔ #06 과제정의 각 필드(도착) |
| 6 | 가치가 정량화됐는가? | **△** | #05 ROI=ΔS×N 구조는 잡힘, 값은 placeholder(τ/s/r/N) → 캠프 실측 시 ✅ (현 단계 △ 정상) |

> **판정 결론** — ✅ 5 + △ 1. 모든 질문이 가리키는 출처 파일이 답을 내므로 **12개 산출물은 한 줄(전략→Pain→근본원인→과제정의→To-Be→KSF→일정)로 모순 없이 꿰인다.** #6의 △는 결함이 아니라 **설계 깊이의 경계**(실측은 캠프 확정) — B-3 후속 작업으로 이월.

## 장표 2 — 교차 정합성 스팟체크 + 마무리 (패키징·발표·피드백 루프)

질문-답 매핑 외에 **파일 간 값이 서로 어긋나지 않는지**(불일치 = 설계 오류) 검산한다.

| 검산 항목 | 대조 대상 | 기대 |
|---|---|---|
| AI 수행 범위 ②~⑥ | #06 적용범위 ↔ #08 🤖 레인 | 동일 |
| 사람 최종 승인 ⑦ | #08 🧑 레인 ↔ #09 조직문화 선결 | 동일 |
| 운영 기준 θ/τ/r | #08 운영기준 ↔ #09 선결 ↔ #10 Test 프로토콜 | 같은 변수·정의 |
| KSF 4관점 ↔ 일정 테마 레인 | #09 4관점 ↔ #10 칸반 행 | 1:1 정렬 |
| placeholder 표기 | 전 파일 (τ/s/r/N·`<담당 임원>`) | 지어낸 수치 없음 |

**B. 마무리 — 피드백 루프 (placeholder 확정 = △→✅ 후속 작업)**

| 확정 대상 | placeholder | 확정 주체 | 반영 위치 |
|---|:---:|---|---|
| Champion(담당 임원) | `<담당 임원>` | 조직 | #06 담당임원 필드 |
| 신뢰도·재정합 임계 | θ, τ | 엔지니어(캠프) | #08 운영기준 · #09 선결 · #10 Test |
| As-Is/To-Be 소요시간 | s, s′ | 실측(Phase 0~1) | #07 · #08 · #05 ROI |
| 재시도 임계·처리량 | r, N | 엔지니어·파일럿 | #08 · #05 수익성 |

> **패키징·발표·루프** — #00 README가 11산출물을 한 묶음으로 인덱싱(인덱스↔파일 일치=패키징 검증) / 각 산출물(표·플로우·칸반)을 1산출물=1~2장 장표로 변환(PPTX 호환). **업데이트 루프**: 캠프 피드백 → placeholder 입력 → 해당 #0N 수정 → #6 재검산 → #00 last_updated 갱신 ⇒ #6 △→✅.
