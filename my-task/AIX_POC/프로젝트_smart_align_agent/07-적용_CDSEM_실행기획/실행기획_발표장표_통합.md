# CD-SEM Align Agent · 실행기획 발표 장표 통합 (07~11, 10매)

> 07 As-Is · 08 To-Be · 09 KSF·제약 · 10 개발일정 · 11 Review 의 발표 장표(각 2매)를 한 덱으로 통합. `## 장표 N` = 슬라이드 1장.

## 장표 1 — [07] As-Is 프로세스 정의 · Activity Flow (Work / Event)


**프로세스 정의** — *CD-SEM Recipe Creation 셋업*: 측정 의뢰(Event)를 받아 신규 Recipe를 만들고, 시험측정에서 발생하는 패턴 오인식·좌표 오류를 통과 기준까지 수정·재측정하여 Recipe DB에 등록하는 업무. **현재 ①~⑦ 전 단계를 🧑 사람이 수동 수행** — 병목 ③·④·⑥, **최대 병목 ⑥**(시험측정↔수정 반복 루프).

```
 ◆[Start] 측정 의뢰서 접수(MIDAS·PPT)
     │
     ▼
 [🧑① 대상확인] → [🧑② 생성·로드] → [🧑③ 패턴등록 ◀병목] → [🧑④ 좌표·박스 ◀병목] → [🧑⑤ 이미징조건]
                                                                                         │
   ┌─────────────────────────────────────────────────────────────────────────────────────┘
   ▼
 [🧑⑥ 시험측정 Try ◀최대병목]  ──◆ 통과 미달 ──▶  [오인식·좌표 수정 → 재측정]
         ▲                                                      │
         └────────────────────[ 반복 루프 ]─────────────────────┘
   │ ◆ 통과 충족
   ▼
 [🧑⑦ 저장·검증]  ──▶  ◆[End] Recipe DB 등록 완료
```

**Work / Event 기록** — 활동(Work, 🧑 박스)은 사람 수행 단위, 이벤트(◆)는 활동의 시작·분기·종료를 일으키는 상태 변화다.

| 구분 | 노드 | 내용 | 비고 |
|---|---|---|---|
| ◆ Event (Start) | 의뢰서 접수 | 공정·소자 엔지니어가 MIDAS에 의뢰서(현 PPT) 등록 | 프로세스 착수 트리거 |
| ▢ Work | ①~⑤ 셋업 | 대상 확인 → 생성·로드 → 패턴 등록 → 좌표·박스 → 이미징 조건 | ③④ 병목, 전부 🧑 |
| ◆ Event (Gateway) | 통과 기준 판정 | 시험측정 결과 통과/미달 분기 | 미달 시 ⑥ 반복 루프 |
| ▢ Work (최대 병목) | ⑥ Try·수정 | 공정 variation 이미지 차이로 오인식·좌표 오류 수정 → 재측정 반복 | `s₆` 최대 |
| ◆ Event (End) | DB 등록 완료 | ⑦ 저장·검증 후 Recipe DB 등록 | 프로세스 종료 |

## 장표 3 — [08] To-Be 프로세스 정의 · Swimlane (🤖 AI / 🧑 Human) + 운영 기준


**프로세스 정의** — As-Is ①~⑦와 **같은 축** 위에서 **①·⑦은 사람(🧑), ②~⑥은 VLM(🤖)이** SEM 화면을 판독하며 GUI를 직접 제어해 연속 수행하고, 임계(θ·τ·r)를 벗어난 예외만 엔지니어에게 핸드오프하는 **Human-in-the-loop**. As-Is 최대 병목 ⑥(반복 수정)을 **VLM 실시간 재정합**으로 정조준한다.

**To-Be Swimlane (🤖/🧑 레인 × 단계 ①~⑦)** — 같은 단계 축에서 ①·⑦만 🧑, ②~⑥은 🤖 연속 자동.

| 레인 ＼ 단계 | ① 의뢰서 | ② 생성·로드 | ③ 패턴 | ④ 좌표·박스 | ⑤ 이미징 | ⑥ 재정합 | ⑦ 저장·승인 |
|---|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| **🧑 Human** | ● 검토·<br>플래깅 | | | | | | ● 최종검증·<br>승인 ◀게이트 |
| **🤖 AI (VLM)** | | ●━ | ━ | ━ | ━ | ━● 실시간<br>재정합 | |

> **흐름·핸드오프** — ② ─▶ ③ ─▶ ④ ─▶ ⑤ ─▶ ⑥ : 하나의 VLM Agent가 SEM 화면을 판독해 GUI를 직접 제어하며 **연속 자동 수행**(⑥은 좌표 shift·패턴 오인식을 실시간 재정합·자동 수정). **예외**(θ·τ 미달 또는 재시도 r 초과) → 🧑 엔지니어 **핸드오프** → 수동 보정 후 **⑦ 승인 게이트**로 복귀. *완전자동화 아님 — Human-in-the-loop.*

**운영 기준 (θ·τ·r·승인 게이트)** — ②~⑥ 자동 수행이 폭주하지 않도록 게이팅한다. 기호는 #09 KSF·05 PoC 측정 변수와 **동일**(설계 일관성).

| 변수 | 의미 | 적용 위치 | 비고 |
|---|---|---|---|
| **θ** | 화면매칭 신뢰도 임계 | ②~⑤ (GUI 인식·제어) | 미달 시 재시도 또는 핸드오프 |
| **τ** | 재정합 정확도 임계 | ⑥ (좌표 재정합·오인식 수정) | #09 KSF#1 수용 기준 = 05 PoC 변수 |
| **r** | 재시도 횟수 임계 | ②~⑥ 공통 | r 초과 = 예외 핸드오프(#09 KSF#3) |
| **승인 게이트** | 사람 승인 없이는 Recipe 등록 불가 | ⑦ | #09 선결조건(⑦ 사람 승인 유지)과 일치 |

## 장표 5 — [09] KSF 매트릭스 (4관점 × 핵심성공요소·장애요인·선결조건·적용기술)

|---|---|---|---|---|
| **업무 프로세스** | ②~⑥ 연속 수행 + 예외 시 기준대로 사람 핸드오프 | ⑥ 좌표 재정합 잔존 오류 · 예외 기준 모호 | 신뢰도 임계 τ·재시도 r 정의 + ⑦ 승인 게이트 확정 | 신뢰도 룰엔진 · 핸드오프 오케스트레이션 |
| **데이터** | SEM 이미지·ref recipe·라벨 충분 확보 | 배율·노이즈 편차 · 라벨 공수 · 사내문서 99% **DRM** | 숙련 엔지니어 라벨 + 스크린샷→VLM 추출 파이프라인 | Qwen3-VL 추출 · BGE-M3 검색 |
