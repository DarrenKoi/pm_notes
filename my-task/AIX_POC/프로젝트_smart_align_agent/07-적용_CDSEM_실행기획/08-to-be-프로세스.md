# 08 · To-Be 업무 프로세스 정의 (Step10 / Design) — VLM GUI-제어 셋업 흐름

> As-Is ①~⑦와 **같은 축** 위에서, ①·⑦은 사람(🧑)이, ②~⑥은 VLM(🤖)이 SEM 화면을 판독하며 GUI를 직접 제어해 연속 수행하고, 임계값을 넘는 예외만 엔지니어에게 핸드오프하는 **Human-in-the-loop 프로세스**를 1장의 플로우차트와 확장 테이블로 못 박는다.

## 왜 필요한가 (Why)

#07에서 그린 As-Is ①~⑦는 전 구간이 사람(🧑) 작업이고, 병목 ③④와 최대 병목 ⑥이 숙련도·작업시간에 묶여 있다 (#07 인용, 출처: [SPEC §2](./_internal/SPEC.md)). To-Be는 이 흐름을 "어느 칸을 AI가, 어느 칸을 사람이 하는가"로 **재정의**해, #06 과제 정의서가 약속한 "AI Agent 대상 업무 ②~⑥"를 실제 프로세스 그림으로 확정한다 (#06 인용).

방법론 틀(02)이 강제하는 첫 번째 점검은 **"AI·사람 역할의 명확한 구분"** 이다 (출처: [원문 07](../../lectures/captures/07-ai-agent-scope-template.md), [원문 10](../../lectures/captures/10-check-and-review.md)). 본 문서는 🤖/🧑 레인을 가진 Swimlane으로 그 구분을 시각화해, 이후 #11 review가 ✅로 검산할 수 있는 근거를 남긴다.

가장 중요한 설계 원칙은 **완전자동화가 아니라는 점**이다. ②~⑥의 연속 자동 수행은 신뢰도 임계(θ·τ)·재시도(r)·승인 게이트라는 **운영 기준** 안에서만 동작하고, 기준을 벗어나면 즉시 사람에게 넘긴다. 이 운영 기준의 변수(θ/τ/r)는 #09 KSF#1·#3 수용 기준 및 [05 PoC 실험설계](../05-적용_CDSEM_PoC실험설계.md)의 측정 변수와 **동일한 기호**여야 하며(설계 일관성), ②~⑥의 레인 전환 경계는 #06 적용범위와 **글자 그대로 일치**해야 한다 — 불일치는 설계 오류다 (출처: [SPEC §3 08 항목](./_internal/SPEC.md)).

## 핵심 산출 (What)

1. **To-Be 플로우차트** — As-Is와 같은 ①~⑦ 축, 🤖/🧑 레인 구분, ②~⑥ 연속 자동 수행 흐름, ⑥의 실시간 재정합 표시, 그리고 예외(임계 초과)→🧑 핸드오프 화살표.
2. **As-Is→To-Be 확장 테이블** — 컬럼: 단계 | To-Be 수행(재정의) | 적용 AI 기술 | 수행주체 | 소요시간(s′) | 사용 시스템 | 운영 기준.
3. **운영 기준 정의** — θ(화면매칭 신뢰도), τ(재정합 정확도 임계), r(재시도 횟수), 승인 게이트. 모두 placeholder이며 #09·05와 기호 공유.

## 작성 결과 (How)

### (1) To-Be 플로우차트 (🤖/🧑 레인)

As-Is(#07)와 동일한 ①~⑦ 가로축. As-Is가 전부 🧑였던 것과 대비해, ②~⑥이 🤖 레인으로 이동한 것이 핵심 변화다 (출처: [원문 07](../../lectures/captures/07-ai-agent-scope-template.md), 04 Step9~10 인용).

```
 ①          ②          ③          ④          ⑤          ⑥           ⑦
의뢰서       Recipe      패턴        좌표·박스    이미징       오인식       저장·
검토·플래깅  생성·로드    등록        ·알고리즘    조건         실시간       검증·
                                                            재정합        승인
─────────────────────────────────────────────────────────────────────────────
🧑 Human    │                                                          │  🧑 Human
(부족정보   │                                                          │  (최종
 플래깅,    │                                                          │   검증·
 필요시)    │                                                          │   승인)
   │        │                                                          │     ▲
   ▼        ▼                                                          │     │
        ┌────────── 🤖 AI Agent (VLM, computer-use) ──────────────┐    │     │
        │  ② ─▶ ③ ─▶ ④ ─▶ ⑤ ─▶ ⑥                                 │ ─▶ │ (게이트)
        │  VLM이 SEM 화면을 판독하고 GUI를 직접 클릭·입력하여        │    │
        │  ②~⑤를 연속 수행 →  ⑥에서 실측 좌표 shift·align 실패·     │    │
        │  패턴 오인식을 실시간 재정합·재align·자동 수정             │    │
        └──────────────────────────────────────────────────────────┘    │
                                       │                                  │
                          예외(신뢰도/정확도 임계 초과:                   │
                          θ·τ 미달 또는 재시도 r 초과)                    │
                                       │                                  │
                                       ▼                                  │
                          🧑 엔지니어 핸드오프 ──────────────────────────┘
                          (사람이 수동 보정 후 ⑦ 승인으로 복귀)

범례:  🤖 AI Agent(VLM) 수행 업무    🧑 Human 수행 업무
원칙:  완전자동화 아님 — ②~⑥ 자동 수행은 운영 기준(θ·τ·r·승인게이트) 안에서만 성립,
       기준 이탈 시 즉시 사람에게 핸드오프 (Human-in-the-loop)
```

- **①🧑** : 의뢰서(MIDAS, 현재 PPT 형태)의 부족정보를 자동 플래깅하고 필요 시 사람이 검토. MIDAS 전산화는 외부 변수이므로 본 과제는 플래깅 선에서 연계 (출처: [SPEC §2](./_internal/SPEC.md)).
- **②~⑥🤖** : 하나의 VLM Agent가 ②부터 ⑥까지를 GUI 제어로 **연속** 수행. 영업 예시처럼 여러 Agent로 쪼개지 않고 단일 흐름으로 묶는다 (04 인용 §"구현 매핑").
- **⑥** : 실측 시작 시 좌표 shift·align 실패·패턴 오인식을 **실시간 재정합·재align·자동 수정** — As-Is 최대 병목을 정조준한 핵심 (#07·#06 인용).
- **⑦🧑** : 엔지니어 최종 검증·승인(승인 게이트). VLM 자가점검 리포트가 보조.
- **예외** : 화면매칭/재정합 신뢰도가 임계(θ·τ) 미달이거나 재시도 r을 초과하면 사람에게 핸드오프.

### (2) As-Is → To-Be 확장 테이블

As-Is 수행·소요시간(s)·시스템은 #07에서 상속받고, To-Be 재정의·적용 AI 기술·수행주체·s′·운영 기준을 채운다. **수치는 전부 placeholder**이며 캠프에서 엔지니어가 확정한다 (출처: [SPEC §1-5](./_internal/SPEC.md)).

| 단계 | To-Be 수행 (재정의) | 적용 AI 기술 | 수행주체 | 소요시간(s′) | 사용 시스템 | 운영 기준 |
|------|---------------------|--------------|----------|--------------|-------------|-----------|
| ① 의뢰서 검토 | 의뢰서 부족정보 자동 플래깅, 필요 시 사람 검토 | **Kimi-K2.5**(텍스트 의뢰서 파싱) + **VLM**(도면·DRM 스크린샷 판독) | 🧑 (필요시) | s′₁ | MIDAS, 부족정보 플래깅 | 입력 필드 충족률 임계, 미달 항목 플래깅 후 ⑦ 보강 |
| ② Recipe 생성·로드 | VLM이 GUI를 조작해 신규 Recipe 생성·웨이퍼/샘플 로드 | **Qwen3-VL** computer-use (화면판독+GUI제어) | 🤖 | s′₂ | CD-SEM 장비 GUI | 화면매칭 신뢰도 θ 이상, 실패 시 재시도 r |
| ③ 패턴 등록 *(구 병목)* | VLM이 SEM 화면 판독해 Align/Addressing 패턴 등록 | **Qwen3-VL** computer-use (화면판독+GUI제어) | 🤖 | s′₃ | CD-SEM 장비 GUI | 패턴 매칭 신뢰도 θ 이상, 미달 시 핸드오프 |
| ④ 좌표·박스 *(구 병목)* | VLM이 EP 좌표·측정박스·알고리즘 자동 설정 | **Qwen3-VL** computer-use (화면판독+GUI제어) | 🤖 | s′₄ | CD-SEM 장비 GUI | 설정값 신뢰도 θ 이상, 룰 위반 시 핸드오프 |
| ⑤ 이미징 조건 | reference 동일 시 VLM이 배율·전압·FOV 자동 적용 | **Qwen3-VL** computer-use (화면판독+GUI제어) | 🤖 | s′₅ | CD-SEM 장비 GUI, Recipe DB(ref) | ref 일치 시 자동, 불일치 시 ⑦ 협의 플래깅 |
| ⑥ 오인식 실시간 재정합 *(구 최대 병목)* | 실측 이미지 기반 좌표 shift·align 실패·오인식을 실시간 재정합·재align·자동 수정 | **Qwen3-VL-30B** 정밀 비전 | 🤖 | s′₆ | CD-SEM 장비 GUI | 재정합 정확도 τ 이상 + 재시도 r 이내, 초과 시 예외 핸드오프 |
| ⑦ 저장·검증·승인 | 엔지니어 최종 검증·승인, Recipe 저장·등록 | 사람 (**VLM 자가점검 리포트** 보조) | 🧑 (최종) | s′₇ | CD-SEM 장비 GUI, Recipe DB | **승인 게이트** — 사람 승인 없이는 등록 불가 |
| 예외 | 임계 초과 단계를 엔지니어에게 핸드오프 → 수동 보정 후 ⑦ 복귀 | 신뢰도 스코어 룰(θ/τ/r 기반 판정) | 🧑 엔지니어 | — | CD-SEM 장비 GUI, 핸드오프 큐 | θ·τ 미달 또는 r 초과 = 핸드오프 트리거 |

> 합계: ΣS′ = s′₁+…+s′₇. As-Is ΣS(#07) 대비 **ΔS = ΣS − ΣS′** 가 #08(본 문서)→#05 수익성(ROI=ΔS×N)·#10 기대효과의 백데이터가 된다 (출처: [SPEC §3 03·05·10 항목](./_internal/SPEC.md)).

### (3) 운영 기준 — 신뢰도 임계·재시도·승인 게이트

②~⑥ 자동 수행이 폭주하지 않도록, 각 단계는 다음 변수로 **게이팅**된다. 변수 기호는 #09 KSF#1·#3 수용 기준 및 [05 PoC](../05-적용_CDSEM_PoC실험설계.md) 측정 변수와 **동일**하다(설계 일관성, 출처: [SPEC §3 08 항목](./_internal/SPEC.md)).

| 변수 | 의미 | 적용 위치 | 비고 |
|------|------|-----------|------|
| **θ** | 화면매칭 신뢰도 임계 | ②~⑤ (GUI 화면 인식·제어) | 미달 시 재시도 또는 핸드오프 |
| **τ** | 재정합 정확도 임계 | ⑥ (좌표 재정합·오인식 수정) | #09 KSF#1 수용 기준 = 05 PoC 측정 변수 |
| **r** | 재시도 횟수 임계 | ②~⑥ 공통 | r 초과 = 예외 핸드오프(#09 KSF#3) |
| **승인 게이트** | 사람 최종 승인 없이는 Recipe 등록 불가 | ⑦ | #09 조직문화 선결조건(⑦ 사람 승인 유지)과 일치 |

**완전자동화 아님(Human-in-the-loop) 박제** : ②~⑥은 θ·τ·r 안에서만 자동이고, ① 플래깅 검토·⑦ 최종 승인·예외 핸드오프는 항상 사람이 통제권을 갖는다. 이 경계가 #09 KSF "선결 조건"(⑦ 사람 승인)과 #02 AI 대체 판정(부분 자동화)과 정렬된다 (#09·#02 인용).
