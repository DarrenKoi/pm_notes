# 10 · 개발 일정 수립 — 발표 장표 (2매)

> [10-개발일정-수립.md](./10-개발일정-수립.md) 산출물을 발표용 2장으로 압축한다. 템플릿 산출물(① 분기 칸반 + ② 게이트 레인 + ③ 변화관리 + ④ 로드맵 노트)을 **장표 1 = 칸반(게이트·변화관리 레인 포함)**, **장표 2 = 게이트 G0~G3 상세 + 오너십·RACI·Kill** 로 나눈다. 소요·인력·τ/s/r/N·`<리드>`는 캠프 확정 placeholder. `## 장표 N` = 슬라이드 1장.

## 장표 1 — 분기 칸반 보드 (Q1~Q4 × 테마, 🚦게이트 · 👤변화관리)

**목표** 1년(2027) · **단위** 분기(Q1~Q4) · **DoD** 운영 이관/제한 파일럿 통과. **Q1에 최대 리스크 PoC를 배치**해 Stage-Gate(G0)로 먼저 친다. 테마 레인은 #09 KSF 4관점과 1:1.

| 테마 (레인) | Q1 — Phase0 PoC<br>(재정합·KSF#1) | Q2 — Phase1<br>(단일 layer ②~⑤) | Q3 — Phase2<br>(⑥ 본격+핸드오프) | Q4 — Phase3<br>(확장·이관) |
|---|---|---|---|---|
| **프로세스**<br>(↔업무) | As-Is/To-Be 확정·임계 θ/τ/r 정의 | ②~⑤ 자동 수행·⑦ 승인 게이트 | ⑥ 재정합+예외 핸드오프 도입 | 운영 이관·SOP·MIDAS 연계 |
| **데이터** | SEM 이미지 수집·숙련자 라벨·hard set | 캡처+VLM 추출 파이프라인·BGE-M3 | 데이터셋 확장·핸드오프 피드백 | 운영 데이터 축적·재학습셋 |
| **AI 모델**<br>(↔시스템) | 재정합 PoC (8B vs 30B 비교) | computer-use 통합(②~⑤)·GUI 연동 | 30B 정밀(⑥)·신뢰도 룰엔진 | 일반화·8B 경량화·layer 확장 |
| **Test** | 섀도우 환경·측정 프로토콜(τ/s/r/N) | 단일 layer E2E 오프라인 | 예외 핸드오프·수용 게이트 | 제한 파일럿(실장비)·회귀 |
| **🚦 게이트** | **G0 Stage-Gate** | G1 | G2 | G3 |
| **👤 변화관리** | 운영팀 협업·엔지니어 신뢰·HITL 승인 UI | ──▶ 전 구간 | ──▶ 상시 | ──▶ (Execution) |

> 범례 — 모델 레인 ②~⑥ = #08 To-Be 적용범위 일치(Q2=②~⑤, Q3=⑥), 불일치=설계오류. **완전자동화 아님** — ⑦ 사람 승인은 Q2부터 끝까지 유지. 변화관리는 분기에 가두지 않는 가로 레인.

## 장표 2 — 🚦 게이트 레인 (G0~G3) + 실행 로드맵 (오너십·RACI·Kill)

분기말 게이트 = **다음 분기 자원 투입 결정점**(일정 아님). **G0가 핵심 안전장치** — 좌표 재정합(KSF#1)이 안 되면 GUI 제어가 완벽해도 ⑥ 자동화는 성립하지 않으므로, Q1 끝에서 싸고 빠르게 판단한다.

| 게이트 | 시점 | 판정 기준 | 통과 → | 미달 → |
|---|:---:|---|---|---|
| **🚦 G0**<br>(Stage-Gate) | Q1말 | Phase0 PoC: P95 오차 ≤ τ **그리고** 오인식 recall ≥ r | Q2 착수<br>(GUI 제어 통합) | **과제 재설계** — fine-tune/전용 vision/hybrid (자원 투입 전 중단) |
| **🚦 G1** | Q2말 | 단일 layer ②~⑤ E2E 오프라인 통과·⑦ 승인 동작 | Q3 착수<br>(⑥ 본격) | layer/패턴 좁혀 진행 + 원인 분석 |
| **🚦 G2** | Q3말 | ⑥ 재정합+예외 핸드오프 수용 게이트(recall/false-alarm) | Q4 착수<br>(확장·이관) | 핸드오프 임계 τ/r 재조정 후 재검증 |
| **🚦 G3** | Q4말 | 제한 파일럿 회귀 통과·운영 SOP 정립 | 운영 이관·확장 결정 | 파일럿 범위 유지·안정화 |

**실행 로드맵 노트 (오너십 · RACI · Kill)**

| 구분 | 내용 |
|---|---|
| **오너십** | Champion=`<담당 임원>`(A) · 워크스트림 리드(R): 프로세스·데이터·AI모델·Test·변화관리 각 `<리드>` |
| **RACI** | **R** 워크스트림 리드 · **A** Champion · **C** 데이터/모델 오너·**보안 승인자**·운영팀 · **I** 현업 엔지니어 |
| **Kill 기준** | 게이트 재설계 **누적 2회 초과 → 재설계 아닌 과제 중단(Kill)**, 결정=Champion (좀비 과제 자원 잠식 방지) |

> 소요·인력·τ/s/r/N·`<담당 임원>`·`<리드>`는 캠프/PoC에서 엔지니어 확정 placeholder — 본 장표는 순서·게이트·오너십 구조만 고정한다.
