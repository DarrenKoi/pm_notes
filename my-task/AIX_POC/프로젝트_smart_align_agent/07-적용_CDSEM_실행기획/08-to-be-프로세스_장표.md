# 08 · To-Be 프로세스 정의 — 발표 장표 (2매)

## 장표 1 — To-Be 프로세스 정의 · Swimlane (🤖 AI / 🧑 Human) + 운영 기준

**프로세스 정의** — As-Is ①~⑦와 **같은 축** 위에서 **①·⑦은 사람(🧑), ②~⑥은 VLM(🤖)이** SEM 화면을 판독하며 GUI를 직접 제어해 연속 수행하고, 임계(θ·τ·r)를 벗어난 예외만 엔지니어에게 핸드오프하는 **Human-in-the-loop**. As-Is 최대 병목 ⑥(반복 수정)을 **VLM 실시간 재정합**으로 정조준한다.

**To-Be Swimlane (🤖/🧑 레인 × 단계 ①~⑦)** — 같은 단계 축에서 ①·⑦만 🧑, ②~⑥은 🤖 연속 자동.

| 레인 ＼ 단계 | ① 의뢰서 | ② 생성·로드 | ③ 패턴 | ④ 좌표·박스 | ⑤ 이미징 | ⑥ 재정합 | ⑦ 저장·승인 |
|---|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| **🧑 Human** | ● 검토·<br>플래깅 | | | | | | ● 최종검증·<br>승인 ◀게이트 |
| **🤖 AI (VLM)** | | ●━ | ━ | ━ | ━ | ━● 실시간<br>재정합 | |

> **흐름·핸드오프** — ② ─▶ ③ ─▶ ④ ─▶ ⑤ ─▶ ⑥ : 하나의 VLM Agent가 SEM 화면을 판독해 GUI를 직접 제어하며 **연속 자동 수행**(⑥은 좌표 shift·패턴 오인식을 실시간 재정합·자동 수정). **예외**(θ·τ 미달 또는 재시도 r 초과) → 🧑 엔지니어 **핸드오프** → 수동 보정 후 **⑦ 승인 게이트**로 복귀. *완전자동화 아님 — Human-in-the-loop.*

**운영 기준 (θ·τ·r·승인 게이트)** — ②~⑥ 자동 수행이 폭주하지 않도록 게이팅한다. 기호는 #09 KSF·05 PoC 측정 변수와 **동일**(설계 일관성).

| 변수 | 의미 | 적용 위치 | 비고 |
|---|---|---|---|
| **θ** | 화면매칭 신뢰도 임계 | ②~⑤ (GUI 인식·제어) | 미달 시 재시도 또는 핸드오프 |
| **τ** | 재정합 정확도 임계 | ⑥ (좌표 재정합·오인식 수정) | #09 KSF#1 수용 기준 = 05 PoC 변수 |
| **r** | 재시도 횟수 임계 | ②~⑥ 공통 | r 초과 = 예외 핸드오프(#09 KSF#3) |
| **승인 게이트** | 사람 승인 없이는 Recipe 등록 불가 | ⑦ | #09 선결조건(⑦ 사람 승인 유지)과 일치 |

## 장표 2 — As-Is → To-Be 확장 테이블

적용 AI는 **사내 LLM만**(Kimi-K2.5·Qwen3-VL), 시스템은 ①(MIDAS) 외 전 단계 CD-SEM 장비 GUI, 운영 기준(θ/τ/r)은 장표 1, 수치는 placeholder.

| 단계 | To-Be 수행 (재정의) | 적용 AI 기술 | 주체 | s′ |
|---|---|---|:---:|:---:|
| ① 의뢰서 검토 | 의뢰서 부족정보 자동 플래깅, 필요 시 사람 검토 | Kimi-K2.5(텍스트 파싱) + VLM(도면 판독) | 🧑 | s′₁ |
| ② Recipe 생성·로드 | VLM이 GUI 조작해 신규 Recipe 생성·로드 | Qwen3-VL computer-use | 🤖 | s′₂ |
| ③ 패턴 등록 *(구 병목)* | SEM 화면 판독해 Align/Addressing 패턴 등록 | Qwen3-VL computer-use | 🤖 | s′₃ |
| ④ 좌표·박스 *(구 병목)* | EP 좌표·측정박스·알고리즘 자동 설정 | Qwen3-VL computer-use | 🤖 | s′₄ |
| ⑤ 이미징 조건 | ref 동일 시 배율·전압·FOV 자동 적용 | Qwen3-VL computer-use | 🤖 | s′₅ |
| ⑥ 실시간 재정합 *(구 최대 병목)* | 좌표 shift·패턴 오인식 실시간 재정합·자동 수정 | Qwen3-VL-30B 정밀 비전 | 🤖 | s′₆ |
| ⑦ 저장·검증·승인 | 엔지니어 최종 검증·승인·Recipe 등록 | 사람 (VLM 자가점검 리포트 보조) | 🧑 | s′₇ |

> ▸ **예외 핸드오프** — θ·τ 미달 또는 재시도 r 초과 → 🧑 엔지니어 핸드오프 → 수동 보정 후 ⑦ 승인 복귀 (운영 기준 θ/τ/r = 장표 1).

> **절감폭 ΔS** — `ΣS′=s′₁+…+s′₇`, As-Is 대비 **ΔS=ΣS−ΣS′** → #05 수익성(ROI≈ΔS×N)·#10 기대효과 백데이터. ΔS 대부분은 ⑥(`s′₆ ≪ s₆`). **#06 스코프 구분**: 06의 *주 400건·5→1분* 은 ⑥ 중 Align fail 1차 PoC(측정 10%) → 1차 PoC ROI=400건×4분(06 종결), 전체 ROI=ΔS×N(#05) 별도 — *1차 PoC ⊂ 전체*.
