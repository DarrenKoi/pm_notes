# 02 · 대상 업무영역 정의 (Step2 Scope Analysis / Discovery)

> 계측(MI) 운영 전체를 L2 컬럼×L3 프로세스 플로우로 가로로 펼치고, 각 L3에 AI 대체 판정을 오버레이해 **Auto Recipe Creation = Recipe Creation(②~⑥)** 을 1차 대상으로 좁혀 확정하고, 업무 정의·L1~L5 break down으로 구체화한다.

> 🔄 **변경 이력 (2026-06-29)**: **R&D 연구소 컨텍스트 정정** — 구 L2 "측정 실행(양산 Run)" 컬럼 삭제, "장비 개발 & 양산 이관" L2 신설. 우리 조직은 R&D 연구소에서 디바이스를 **개발**하며, "양산"은 운영 대상이 아니라 개발 완료 디바이스/장비를 양산으로 **이관**하는 별도 업무다. 따라서 양산 Run은 L2에서 제외하고, 양산 이관은 장비 개발 흐름의 종착점(L2-D ④)으로 흡수했다. ("효율화" 같은 목표성 항목은 L2에서 제외하고 [01 M1 업무 목표](./01-전략일관성-확인.md)로 이관.) 구 L2 4분할 "엔지니어 확정 2026-06-24" → 본 정정으로 **재확정 2026-06-29**.

## 왜 필요한가 (Why)

[01-전략일관성-확인](./01-전략일관성-확인.md)에서 "CD-SEM **Auto Recipe Creation**"이 조직목표(R&D 디바이스 개발 지연 최소화)로부터 핵심업무(MI Recipe Solution 제공·측정 장비 간 Skew Zero·MI 장비 가동률 극대화)를 거쳐 연역된 과제임을 추적성으로 증명했다. 다만 그 과제의 대상 업무(Recipe Creation)가 **계측 운영이라는 더 큰 업무 지도의 어디에 위치하는지, 그리고 그 주변 업무 중 무엇을 AI로 대체할지/말지**를 가리지 않으면 PoC 범위가 흐려진다.

- **"모든 칸에 AI를 넣지 않는다"**: AI Agent는 **반복성·정형성·공수·데이터 가용성**이 큰 지점에만 선별 배치한다 (출처: [원문 06](../../lectures/captures/06-sales-ai-opportunity.md) 해석 포인트, [원문 01 STEP3](../../_가이드/01-기획문서_AX서비스기획.md)).
- **개입 최소 단위는 L5(Task)**: 업무를 L1~L5로 분해해야 AI를 어디에 배치할지 정밀하게 식별할 수 있다 (출처: [원문 05](../../lectures/captures/05-process-level-definition.md) 해석 포인트). 본 문서는 L1~L3까지 가로로 펼쳐 **1차 대상 L3를 선별**하고, L4~L5 분해와 정량화는 [03-painpoint-도출-정량화](./03-painpoint-도출-정량화.md)로 넘긴다.
- **PoC 경계의 합의**: 1차 대상과 차기(2차) 대상을 명시적으로 분리해, 이후 모든 산출물(과제 정의·To-Be·일정)이 동일한 범위 위에서 작성되게 한다.

## 핵심 산출 (What)

1. **L1~L3 가로 레이아웃 다이어그램** — L1=계측(MI) 운영, L2 컬럼 4개, 각 L2 아래로 L3 프로세스 플로우. (포맷 출처: [원문 05](../../lectures/captures/05-process-level-definition.md)의 L1~L5 가로 위계 표)
2. **AI 대체 판정 오버레이 표** — 각 L3에 🤖/🤝/🧑/⚙️ 판정 + 근거(반복성·정형성·공수·데이터가용성 4기준).
3. **범위 결정** — 1차 대상 L3와 차기 대상 L3를 명문화.

> ✅ **L2 4분할은 엔지니어 확정 항목이다 (2026-06-24 확정 → 2026-06-29 R&D 컨텍스트 재확정).** 아래 L2-A~D 구획과 각 L3 목록은 도메인 엔지니어 검토를 거쳐 **계측(MI) 운영의 프로세스 위계로 확정**되었고, 이번 R&D 컨텍스트 정정으로 구 "측정 실행(양산 Run)"을 삭제하고 "장비 개발 & 양산 이관"을 신설해 재확정했다. 셋업(②~⑥, A 영역의 Recipe Creation)은 As-Is로 확정돼 있고, L2/L3 경계도 동일하게 확정. (출처: [03 §2 L1~L5](../03-적용_CDSEM기획.md) + 엔지니어 확정 2026-06-24·재확정 2026-06-29)

## 작성 결과 (How: L2×L3 플로우 + 대체 판정)

### A. L1~L3 가로 레이아웃 (Process Level Map)

L1 = **계측(MI) 운영** 하나. 그 아래 L2 4개 컬럼을 좌→우로 두고, 각 컬럼 안에서 L3가 위→아래(또는 좌→우) 흐름을 갖는다. (포맷: [원문 05](../../lectures/captures/05-process-level-definition.md) / 배치 마킹 개념: [원문 06](../../lectures/captures/06-sales-ai-opportunity.md))

```

   └─────────┬───────────┘   └─────────┬───────────┘   └─────────┬───────────┘   └─────────┬───────────┘
             │ (L3 flow)               │ (L3 flow)               │ (L3 flow)               │ (L3 flow)
             ▼                         ▼                         ▼                         ▼
L3   ① 의뢰 접수             ① BM/PM                   ① Recipe 준수 모니터링 ◇ ① 로드맵 매칭
        │                       │                         │                        │
        ▼                       ▼                         ▼                        ▼
     ② Recipe Creation ★★    ② 장비 상태 모니터링      ② 계측 파라미터 최적화   ② 장비 개발 (BP社)
        │  (= 셋업 ②~⑥)       │                         │                        │
        ▼                       ▼                         ▼                        ▼
     ③ 검증·등록             ③ 캘리브레이션            ③ 데이터 분석·리포트     ③ 장비 도입
        │                                                                          │

  ★  = 1차 PoC 대상 L2 (A. Recipe Management)
  ★★ = 1차 PoC 대상 L3 (Recipe Creation, 내부 셋업 ②~⑥)
  ◇  = 차기(2차) 대상 L2 (C. 계측 데이터·모니터링, 대표 L3 = ① Recipe 준수 모니터링)
```

- L2-A의 L3②(Recipe Creation)는 03의 SOP에서 **[B] Recipe 셋업 ①~⑦**에 해당하며, 그중 AI 수행 범위는 **②~⑥**이다 (출처: [03 §2](../03-적용_CDSEM기획.md), 셋업 ①~⑦ 중 병목 ③④⑥·최대병목 ⑥; AI 수행=②~⑥, 사람=① 검토·⑦ 최종승인).
- L2-C의 L3①(Recipe 준수 모니터링)은 등록 Recipe가 실제 계측에서 의도대로 적용·준수되는지 추세 감시하는 구간으로, 사람 주기점검·야간/주말 의존으로 **운영 공백**이 큰 차기(2차) 대상이다 (출처: [03 §2 Phase1](../03-적용_CDSEM기획.md), 모니터링 공백).
- L2-B(장비 운영/관리)·L2-D(장비 개발 & 양산 이관)는 물리 보전·외부 협력(BP社)·도입/이관 행정 중심이라 계측 Recipe 자동화 범위 밖이다. 특히 **양산은 우리 조직(R&D 연구소)의 운영 대상이 아니라, 개발 완료된 디바이스/장비를 양산으로 이관**하는 업무로 L2-D의 종착점(④ 양산 이관)에 흡수했다 (R&D 컨텍스트 정정 — 구 "측정 실행(양산 Run)" 컬럼 삭제).

### B. AI 대체 판정 오버레이

**판정 범례**

| 아이콘 | 의미 |
|--------|------|
| 🤖 | AI 대체 (반복·정형·고공수, AI가 주수행) |
| 🤝 | 부분 대체 (AI 보조 + 사람 판단 병행, HITL) |
| 🧑 | 사람 (판단·승인·물리작업, AI 부적합) |
| ⚙️ | 기존 자동 (이미 장비/시스템이 자동화, 신규 AI 불필요) |

**판정 기준** = 반복성 · 정형성 · 공수 · 데이터 가용성 (출처: [원문 06](../../lectures/captures/06-sales-ai-opportunity.md) "반복성·정형성·공수가 큰 Task 지점 선별"). 아래 표의 L2 구획·L3 목록은 엔지니어 확정(2026-06-24)·R&D 컨텍스트 재확정(2026-06-29).

### C. 범위 결정 (Scope Decision)

```
1차 PoC 대상  =  L2-A › L3② Recipe Creation  (내부 셋업 ②~⑥)
                 └ 근거: 반복성·정형성·공수 最大 + SEM 이미지 데이터 이미 가용
                 └ 최대 병목 ⑥(오인식 수정 반복) 집중 → VLM GUI-제어 + 실시간 좌표 재정합

차기(2차) 대상 =  L2-C › L3① Recipe 준수 모니터링  (계측 데이터·모니터링)
                 └ 근거: 운영 공백(사람 주기점검·야간/주말 의존), 데이터 가용성 高
                 └ 1차 안정화 후 Passive→Active 확장 단계에서 착수

범위 밖(이번)  =  L2-B(장비 운영/관리: 물리 보전) · L2-D(장비 개발 & 양산 이관: 외부협력·행정)
                 · A①/A③/A④(사람 판단·승인·후속 추적)
```

- **1차 대상 = L2-A › L3② Recipe Creation(②~⑥) = Auto Recipe Creation**: AI(🤖)가 SEM 화면을 판독하고 GUI를 직접 제어해 Recipe를 생성하며, 실측 시작 시 좌표 shift·align 실패·오인식을 실시간 재정합한다. ①(검토)·⑦(최종승인)은 사람(🧑) 유지 — **완전자동화 아님, Human-in-the-loop** (출처: [SPEC §0](./_internal/SPEC.md) 고정 과제 정의).
- **차기 대상 = L2-C › L3① Recipe 준수 모니터링**: 1차 셋업 자동화가 안정화된 뒤, 운영 공백(야간/주말 모니터링)을 메우는 2차 확장 후보.
- **범위 밖 = L2-B·L2-D**: 장비 운영/관리는 물리 보전·캘리브레이션 중심, 장비 개발 & 양산 이관은 외부 협력(BP社)·도입/이관 행정 중심으로 계측 Recipe 자동화 대상이 아니다. 양산 이관은 R&D 개발 완료품의 종착 업무다.
- L2 4분할과 각 L3 경계는 **엔지니어 확정(2026-06-24) → R&D 컨텍스트 재확정(2026-06-29)** — 계측(MI) 운영 프로세스 위계로 고정. 이후 산출물(과제 정의·To-Be·일정)은 이 범위 위에서 작성된다.

### D. 업무 정의 & L1~L5 프로세스 Break Down (Auto Recipe Creation)

범위가 좁혀진 1차 대상을 **하나의 AI 과제(Auto Recipe Creation)** 로 정의하고, 계측(MI) 운영 위계(L1~L5) 안에서 AI 배치 지점을 못박는다. 이 정의가 이후 #03 Pain 정량화·#06 과제 정의서·#08 To-Be의 공통 기준이 된다.

#### 업무 정의 — Auto Recipe Creation

- **한 줄(Why/What)**: CD-SEM에서 신규 측정 대상 Recipe를 **VLM이 SEM 화면을 보며 GUI를 직접 조작(computer-use)으로 생성·등록**한다. 사람은 예외만 개입(HITL).
- **Input**: 측정 의뢰서(도면·측정위치·파라미터·측정방식), reference recipe, 웨이퍼/샘플, SEM 실시간 화면
- **Output**: 검증 완료된 측정 Recipe(저장·등록)
- **To-Be 경계**:
  - 🤖 **AI** = 셋업 ②~⑥ GUI 조작 · 좌표/align 재정합 · 오인식 수정
  - 🧑 **Human** = ① 의뢰서 검토(필요시) · ⑦ 최종 검증 · 예외 핸드오프
- **범위 밖**: 장비 운영/관리(물리 보전·캘리브레이션) · 장비 개발 & 양산 이관(외부 협력·도입/이관 행정) · 계측 데이터·모니터링(차기)

> 이 과제는 위 [01 연계표](./01-전략일관성-확인.md)의 핵심업무 중 ①~③에 동시에 기여한다 — ① MI Recipe Solution 제공(솔루션 본체), ② 측정 장비 간 Skew Zero(일관 기준 → 숙련도·장비 편차 제거), ③ MI 장비 가동률 극대화(24h 무중단 셋업 → 유휴시간 축소). 핵심업무 ④(장비 개발·양산 이관)는 추적성 표기용으로 ARC 범위 밖이다.

#### L1~L5 프로세스 Break Down

```
L1  계측(MI) 운영
 └ L2  CD-SEM Recipe Management              (= 핵심업무 ① MI Recipe Solution 제공)
     └ L3  Recipe Creation
         └ L4  [A] 접수 ─▶ [B] 셋업 ①~⑦ ─▶ [C] 검증·등록 ─▶ [D] 후속 모니터링
                                │
                                └ L5 (AI 배치) ──────────────────────────────────
                                    🤖 ③ 패턴/Align 등록
                                    🤖 ④ 좌표·측정박스
                                    🤖 ⑥ 오인식 수정      ← 최대 병목 · PoC 핵심
                                    🔜 C① Recipe 준수 모니터링  ← 차기 (L2-C)
                                    🤝 A② 의뢰서 플래깅   ← 보조
```

| 레벨 | 내용 |
|------|------|
| **L1** 계측(MI) 운영 | Mega Process |
| **L2** CD-SEM Recipe Management | = 핵심업무 ① **MI Recipe Solution 제공** |
| **L3** Recipe Creation | 의뢰 접수~셋업~검증·등록~후속 모니터링 한 흐름 (양산 Run 제외 — R&D 컨텍스트) |
| **L4** [A]접수 → [B]셋업①~⑦ → [C]검증·등록 → [D]후속 모니터링 | Activity |
| **L5** (AI 배치) | ③ 패턴/Align 등록 · ④ 좌표·측정박스 · **⑥ 오인식 수정(최대 병목, PoC 핵심)** · C① Recipe 준수 모니터링(차기, L2-C) · A② 의뢰서 플래깅(보조) |

#### PoC 위치

```
Auto Recipe Creation  (셋업 ②~⑥ 전체)
        └ 1차 PoC = Align fail 대응 자동화
              = Recipe Creation 내부 ③(Align 등록) · ⑥(실측 시 Align 실패·오인식 자동 보정)
                  └ 실측 시 좌표 shift·align 실패·오인식을
                    VLM이 SEM 이미지로 실시간 재정합·수정
        ※ 선정 근거: Align fail 수정 ≈ ⑥ 오측/인식 실패 수정 (큰 틀 같은 개념)
                   + align image 재등록(recipe tuning)은 비교적 간단 → PoC 진입점
```

Auto Recipe Creation(셋업 ②~⑥ 전체) 안에서 **1차 PoC = Align fail 대응 자동화(= Align Agent) = ⑥(+③④)** 로 좁힌다. ⑥은 As-Is 최대 병목(오인식 수정 반복)이자 SEM 이미지 데이터가 이미 가용한 지점이라 PoC 가치·실현성이 가장 높다 (병목 ③④⑥·최대병목 ⑥은 엔지니어 확정 — [SPEC §2](./_internal/SPEC.md)).
