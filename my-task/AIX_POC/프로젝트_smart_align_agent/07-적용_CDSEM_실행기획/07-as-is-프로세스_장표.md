## 장표 1 — As-Is 프로세스 정의 · Activity Flow (Work / Event)

**프로세스 정의** — *CD-SEM Recipe Creation 셋업*: 측정 의뢰(Event)를 받아 신규 Recipe를 만들고, 시험측정에서 발생하는 패턴 오인식·좌표 오류를 통과 기준까지 수정·재측정하여 Recipe DB에 등록하는 업무. **현재 ①~⑦ 전 단계를 🧑 사람이 수동 수행** — 병목 ③·④·⑥, **최대 병목 ⑥**(시험측정↔수정 반복 루프).

```
 ◆[Start] 측정 의뢰서 접수(MIDAS·PPT)
     │
     ▼
 [🧑① 대상확인] → [🧑② 생성·로드] → [🧑③ 패턴등록 ◀병목] → [🧑④ 좌표·박스 ◀병목] → [🧑⑤ 이미징조건]
                                                                                         │
   ┌─────────────────────────────────────────────────────────────────────────────────────┘
   ▼
 [🧑⑥ 시험측정 Try ◀최대병목]  ──◆ 통과 미달 ──▶  [오인식·좌표 수정 → 재측정]
         ▲                                                      │
         └────────────────────[ 반복 루프 ]─────────────────────┘
   │ ◆ 통과 충족
   ▼
 [🧑⑦ 저장·검증]  ──▶  ◆[End] Recipe DB 등록 완료
```

**Work / Event 기록** — 활동(Work, 🧑 박스)은 사람 수행 단위, 이벤트(◆)는 활동의 시작·분기·종료를 일으키는 상태 변화다.

| 구분 | 노드 | 내용 | 비고 |
|---|---|---|---|
| ◆ Event (Start) | 의뢰서 접수 | 공정·소자 엔지니어가 MIDAS에 의뢰서(현 PPT) 등록 | 프로세스 착수 트리거 |
| ▢ Work | ①~⑤ 셋업 | 대상 확인 → 생성·로드 → 패턴 등록 → 좌표·박스 → 이미징 조건 | ③④ 병목, 전부 🧑 |
| ◆ Event (Gateway) | 통과 기준 판정 | 시험측정 결과 통과/미달 분기 | 미달 시 ⑥ 반복 루프 |
| ▢ Work (최대 병목) | ⑥ Try·수정 | 공정 variation 이미지 차이로 오인식·좌표 오류 수정 → 재측정 반복 | `s₆` 최대 |
| ◆ Event (End) | DB 등록 완료 | ⑦ 저장·검증 후 Recipe DB 등록 | 프로세스 종료 |

## 장표 2 — As-Is 각 Activity 상세 (소요시간 · 사용 시스템)

소요시간(`s₁..s₇`)·사용 시스템을 고정 — 백데이터(#08·#05·#10)의 *before* 값. `s₆`(⑥ 반복) 최대, 합계 `ΣS` = 1건당 리드타임.

| 단계 (Activity) | 설명 | 소요시간 | 사용 시스템 |
|---|---|---|---|
| 🧑 ① 측정 대상 확인 | 도면·스펙 검토로 측정 레이어/패턴 파악 | `s₁` | MIDAS(의뢰서) |
| 🧑 ② Recipe 생성·로드 | 신규 Recipe 생성, 웨이퍼/샘플 로드 | `s₂` | CD-SEM 장비 GUI |
| 🧑 ③ 패턴 등록 *(병목)* | Alignment/Addressing 포인트 수동 등록 | `s₃` | CD-SEM 장비 GUI |
| 🧑 ④ 좌표·박스 설정 *(병목)* | 측정 포인트(EP) 좌표·박스·알고리즘 수동 설정 | `s₄` | CD-SEM 장비 GUI |
| 🧑 ⑤ 이미징 조건 설정 | 배율·전압·FOV 등 설정 (ref 없으면 협의) | `s₅` | 장비 GUI / Recipe DB |
| 🧑 ⑥ 시험측정·오인식 수정 *(최대 병목)* | 공정 variation 이미지 차이로 패턴 실패·좌표 오류 → 수정 → 재측정 **반복 루프** | `s₆` *(최대)* | CD-SEM 장비 GUI |
| 🧑 ⑦ 저장·검증 | Recipe 저장·등록 → 검증 (누락 시 후공정 전파 위험) | `s₇` | 장비 GUI / Recipe DB |
| **합계** | **As-Is 셋업 리드타임 (1건당)** | **`ΣS`** | — |

> **백데이터 연결** — 소요시간 → #08 *before* 값 · #05 ΔS=ΣS−ΣS′ As-Is 항 · #10 기대효과 근거 / 사용 시스템 → #09 제약(장비 SW 버전 의존) 출처. **#06 스코프 구분**: 06의 *주 400건·5분* 은 ⑥ 중 Align fail 1차 PoC 슬라이스(측정의 10%) 실측, 본 `s₆`/`ΣS` 는 전체 셋업 1건 시간 → 곱하지 않고 *1차 PoC ⊂ 전체 셋업* 포함관계로 별도 계산.
