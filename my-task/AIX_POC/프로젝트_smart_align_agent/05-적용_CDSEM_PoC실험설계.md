# AIX 적용 사례 — CD-SEM 좌표 재정합 PoC 실험 설계 (KSF#1 검증)

> [04 기술문서](./04-적용_CDSEM기술.md)에서 **1순위 선결 조건(KSF#1)** 으로 못 박은 *"VLM 좌표 재정합 정확도가 엔지니어 수용 기준 이상"* 을 실제로 검증하기 위한 실험 설계서. 04의 개발 일정 **Phase 0**에 해당하며, 통과해야 Phase 1(GUI 제어 통합)으로 넘어간다.

## 왜 이 실험이 먼저인가? (Why)

04의 To-Be는 두 개의 미검증 가정을 동시에 담고 있다 — (가정 A) VLM이 GUI를 직접 제어한다, (가정 B) VLM이 SEM 이미지를 보고 좌표/align을 재정합한다. 둘을 한 번에 검증하면 실패 원인을 분리할 수 없다.

본 PoC는 **가정 B만 격리**한다. 이유:

- **최대 리스크 우선** — ⑥ 오인식 수정의 본질은 "좌표를 맞히는 능력"이다. 이게 안 되면 GUI 제어가 완벽해도 과제는 성립하지 않는다. (No-Regret: 먼저 깨질 곳을 먼저 친다)
- **실 장비 영향 0** — GUI 제어를 빼면 실 장비·웨이퍼를 건드리지 않고 *오프라인 이미지→좌표* 만으로 검증 가능. 안전·비용·승인 부담 최소.
- **싸고 빠른 판단 게이트** — 정확도가 수용 기준 미달이면 GUI 제어 개발에 자원을 쓰기 전에 과제를 재설계한다.

## 무엇을 검증하는가 (What)

### 검증 가설

- **H1 (대립가설)**: 사내 VLM(Qwen3-VL)은 SEM 이미지에서 정렬 패턴과 측정 포인트 좌표를, 실측 시 발생하는 shift/align 실패 상황에서도 **엔지니어 수용 기준(τ) 이내로 재정합**할 수 있다.
- **H0 (귀무가설)**: VLM의 좌표 재정합 오차가 수용 기준을 넘어, ⑥ 자동화에 쓸 수 없다.

### 측정 대상 (03·04에서 지목한 집중 근본원인)

| 대상 | 정의 | 04 연결 |
|------|------|---------|
| **(a) 패턴 인식** | SEM 화면에서 Alignment/Addressing 패턴 위치를 찾는 능력 | 근본원인 (a) 패턴 인식 실패 |
| **(b) 측정박스·좌표** | 측정 포인트(EP) 좌표·측정박스 위치를 제시하는 능력 | 근본원인 (b) 측정박스·좌표 오류 |
| **(c) 오인식 검출** | "이 정합/측정이 틀렸다"를 스스로 판단하는 능력 (⑥의 핵심) | ⑥ 오인식 수정 반복 |

### 성공 기준 (수용 기준 — 임계값은 엔지니어 협의)

> 구체 수치(τ, r, N)는 측정박스 허용 범위·공정 스펙에 의존하므로 **엔지니어가 확정**한다. 본 문서는 기준의 *형식*만 고정한다.

- **정확도**: 예측 좌표와 정답 좌표 간 오차의 **P95 ≤ τ** (τ = 허용 측정박스/정렬 마진 기반).
- **재정합 성공률**: shift/align 실패 케이스에서 오차 ≤ τ로 복구한 비율 **≥ s%**.
- **오인식 검출**: **recall ≥ r** (놓치면 불량 측정 유발 → 높게) + **false-alarm rate ≤ f** (너무 잦은 사람 호출은 효용 상쇄).

## 어떻게 검증하는가 (How)

### 1. 실험 구성

| 요소 | 설계 |
|------|------|
| **데이터셋** | ① 기존 등록 Recipe + 대응 SEM 이미지 = 정답(ground-truth, 숙련 엔지니어 라벨) / ② 과거 ⑥에서 실제 오인식·shift가 났던 케이스 = **hard set** / layer별 균형, 최소 N건(N=엔지니어 협의) |
| **모델** | 사내 **Qwen3-VL-8B vs 30B** 비교 (정확도/latency trade-off 확인) |
| **비교군(baseline)** | ① 장비 기본 템플릿 매칭 성공률(이걸 넘어야 의미) / ② 숙련 엔지니어 수동 재정합(정확도 상한 reference) |
| **출력 방식** | VLM grounding 프롬프트로 정규화 좌표/bbox 출력 → 픽셀(또는 장비 좌표계)로 역변환 후 정답과 비교 |

### 2. 평가 메트릭 (정량)

1. **좌표 오차 분포** — median·P95 (px 또는 nm 환산). 핵심 지표.
2. **재정합 성공률** — shift/align 실패 케이스에서 오차 ≤ τ 복구 비율.
3. **오인식 검출 성능** — precision/recall, false-alarm rate.
4. **자율 처리율 vs 핸드오프율** — 예외 임계 적용 시 사람 개입 없이 통과한 비율(KSF#3 연결).
5. **모델 비교·latency** — 8B vs 30B 정확도·응답시간.

### 3. 실험 절차

```
1) 데이터 수집·라벨링: 기존 Recipe·SEM 이미지에서 정답 좌표 추출 (캡처+VLM 추출, 익명화)
2) hard set 구성: 과거 오인식/shift 케이스 분리
3) 추론: Qwen3-VL(8B·30B)에 이미지 입력 → 좌표·오인식 판단 출력
4) 정답 대조: 오차 분포·성공률·검출 성능 산출
5) baseline 대비: 템플릿 매칭·엔지니어 기준과 비교
6) 게이트 판정 (아래)
```

### 4. 실험 환경 (제약 반영)

- **오프라인·섀도우** — 실 장비/웨이퍼 영향 0. 이 PoC는 *이미지→좌표* 만 검증하고 **실제 GUI 제어는 범위 밖**(Phase 1).
- **사내 엔드포인트만** — 외부 LLM 차단, Qwen3-VL OpenAI-호환 내부 API 사용.
- **DB 로컬 미가용** — 코드 검증은 import·문법 수준, 데이터는 파일 기반.
- **DRM** — 사내 문서/이미지 99% DRM → 스크린샷+VLM 추출이 기본, 익명화 후 사용.

### 5. 판단 게이트 (Phase 0 → 다음)

| 결과 | 조건 | 다음 액션 |
|------|------|-----------|
| **PASS** | P95 오차 ≤ τ **그리고** 오인식 recall ≥ r | Phase 1 진행 — GUI 제어 통합 |
| **CONDITIONAL** | 일부 layer/패턴만 충족 | 충족 범위로 적용 좁혀 진행 + 미충족 원인 분석 |
| **FAIL** | 수용 기준 미달 | 과제 재설계 — fine-tune / 전용 vision 모델 / hybrid(템플릿+VLM) 검토 |

## 리스크 & 한계

- **VLM 좌표 정밀도** — 범용 VLM의 grounding이 계측 수준 정밀도에 못 미칠 수 있음(이 PoC가 검증하려는 바로 그 점). FAIL 시 fine-tune·전용 모델이 대안.
- **정답 라벨 품질** — ground-truth가 숙련 엔지니어 판단에 의존 → 라벨 합의 기준 필요.
- **격리의 한계** — GUI 제어를 뺐으므로, PASS여도 Phase 1에서 *제어 안정성*은 별도 검증 필요(가정 A).
- **데이터 대표성** — hard set이 실제 오인식 분포를 충분히 반영해야 일반화 가능.
