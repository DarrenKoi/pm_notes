# 08 · To-Be 프로세스 정의 — 템플릿

> 한 줄: #07에서 그린 As-Is 흐름을 "어느 단계를 AI가, 어느 단계를 사람이 하는가"로 재정의해, **🤖/🧑 레인을 가진 1장의 To-Be 플로우차트 + 확장 테이블**로 미래 업무 모습을 못 박는다.

## 이 단계가 무엇인가 (Why)

To-Be 정의는 "AI를 어디에 끼울까"가 아니라 **As-Is 흐름을 같은 축 위에서 재배치**하는 작업이다. New AI Design Camp 방법론은 To-Be를 그릴 때 **AI Agent 수행 / Human 수행을 한 장에 시각화**하라고 요구한다 — 이것이 Review 단계(#11)의 첫 점검 항목 "AI·사람 역할의 명확한 구분"을 실제로 작성하는 도구다 (출처: [원문 07 — AI Agent 대상영역 분석 Template](../source/07-ai-agent-scope-template.md), [방법론 §To-Be Swimlane](../02-기술문서_AI과제정의구현.md)).

이 단계 산출물은 #06 과제 정의서가 약속한 "AI Agent 대상 범위"를 **실제 프로세스 그림으로 확정**하고, 각 단계의 To-Be 소요시간(s′)을 채워 #07 As-Is 합계(ΣS) 대비 절감폭(ΔS)을 #05 적정성·#10 기대효과로 넘기는 백데이터가 된다.

> **가장 중요한 설계 원칙: 완전자동화가 아니다.** AI가 연속 수행하는 구간도 신뢰도 임계·재시도·승인 게이트라는 **운영 기준** 안에서만 동작하고, 기준을 벗어나면 즉시 사람에게 넘긴다(Human-in-the-loop). 사람 개입점이 한 곳도 없는 To-Be는 거의 항상 설계 오류다.

> ⚠️ **보강 자가점검 — AI 안전·책임소재(필수)**: AI가 시스템·설비를 **직접 제어**하는 To-Be라면, 운영 기준 테이블에 **안전 인터록·롤백·사고 책임자** 행을 반드시 추가하라. 오작동이 물리적 사고로 이어질 때의 차단·되돌림·책임이 비면 현장이 거부한다. 제어가 없는 과제는 명시적으로 "해당 없음" 표기. (→ [보강노트](./보강노트_놓치기쉬운-항목.md) 필수 #3)

## 작성할 산출물 (What)

### ① To-Be 플로우차트 (🤖/🧑 레인, 빈 양식)

> 작성 가이드: #07 As-Is와 **같은 단계 축**(①②③…)을 그대로 쓴다. 달라지는 것은 "어느 박스가 어느 레인으로 가는가"다. AI가 하는 단계는 🤖 레인, 사람이 하는 단계는 🧑 레인에 둔다. 그리고 운영 기준을 못 넘는 예외가 🤖 레인 → 🧑 레인으로 빠지는 **핸드오프 화살표** 자리를 반드시 그린다.

```
 ①          ②          ③     …      ⓝ
〈단계1〉    〈단계2〉    〈단계3〉      〈단계n〉
────────────────────────────────────────────────
🧑 Human   │                          │  🧑 Human
〈사람이    │                          │ 〈사람이
 하는 진입  │                          │  하는 마지막
 단계〉     │                          │  단계: 승인〉
   │       │                          │     ▲
   ▼       ▼                          │     │
       ┌──── 🤖 AI Agent ───────────┐  │     │ (승인 게이트)
       │  ② ─▶ ③ ─▶ … ─▶ ⓝ-1        │ ─▶│
       │ 〈AI가 연속 수행하는 구간〉   │  │
       └────────────────────────────┘  │
                     │                  │
        예외(운영 기준 미달: 〈θ/τ 미달 또는 재시도 r 초과〉)
                     │                  │
                     ▼                  │
            🧑 〈담당자〉 핸드오프 ──────┘
            (사람이 수동 보정 후 〈승인 단계〉로 복귀)

범례:  🤖 AI Agent 수행 단계      🧑 Human 수행 단계
원칙:  완전자동화 아님 — AI 자동 수행은 운영 기준(〈θ·τ·r·승인게이트〉) 안에서만 성립,
       기준 이탈 시 즉시 사람에게 핸드오프 (Human-in-the-loop)
```

각 자리에 무엇을 그리나:

- **🧑 진입/마무리 레인** = 사람이 시작점(입력 검토·플래깅)과 끝점(최종 검증·승인)을 잡는다. 보통 양 끝은 사람이 통제권을 갖는다.
- **🤖 연속 구간** = AI가 끊김 없이 여러 단계를 수행하는 묶음. 여러 Agent로 쪼갤지 단일 흐름으로 묶을지는 조직 상황에 맞춰 결정한다.
- **예외 핸드오프 화살표** = 운영 기준(아래 ③)을 못 넘는 케이스가 사람에게 빠지는 경로. 이 화살표가 없으면 "완전자동화로 그린 것"이고, 그것이 가장 흔한 실수다.

### ② As-Is → To-Be 확장 테이블 (빈 양식)

> 작성 가이드: As-Is 수행·소요시간(s)·사용 시스템은 #07에서 상속받고, To-Be 재정의·적용 AI 기술·수행주체·s′·운영 기준을 채운다. 행은 단계 수만큼. **수치는 모두 placeholder**(s′=To-Be 소요시간)로 두고 캠프에서 담당자가 확정한다.

| 단계 | To-Be 수행 (재정의) | 적용 AI 기술 | 수행주체 🤖/🧑 | 소요시간 s′ | 사용 시스템 | 운영 기준 |
|------|---------------------|--------------|:-------------:|:-----------:|-------------|-----------|
| ① 〈단계명〉 | 〈AI/사람이 무엇을 하나〉 | 〈적용 기술 보기에서 선택〉 | 〈🤖/🧑〉 | s′₁ | 〈시스템〉 | 〈임계·게이트〉 |
| ② 〈단계명〉 | 〈…〉 | 〈…〉 | 〈🤖/🧑〉 | s′₂ | 〈…〉 | 〈θ 이상, 미달 시 재시도 r〉 |
| ③ 〈단계명〉 | 〈…〉 | 〈…〉 | 〈🤖/🧑〉 | s′₃ | 〈…〉 | 〈…〉 |
| … | 〈…〉 | 〈…〉 | 〈🤖/🧑〉 | s′… | 〈…〉 | 〈…〉 |
| 예외 | 임계 초과 단계를 담당자에게 핸드오프 → 수동 보정 후 〈승인 단계〉 복귀 | 〈θ/τ/r 기반 판정 룰〉 | 🧑 〈담당자〉 | — | 〈핸드오프 큐〉 | 〈θ·τ 미달 또는 r 초과 = 핸드오프 트리거〉 |

> 합계: ΣS′ = s′₁+…+s′ⓝ. As-Is ΣS(#07) 대비 **ΔS = ΣS − ΣS′** 가 #05 수익성(ROI ≈ ΔS×N)·#10 기대효과의 백데이터다. (placeholder: s′=To-Be 소요시간, N=처리량)

**📌 "적용 AI 기술" 보기** (해당 칸에 무엇을 적나) — 조직 가용 모델로 대체:

| 기술 유형 | 무슨 일을 하나 | 어디에 쓰나 |
|-----------|----------------|-------------|
| 생성형 텍스트 LLM | 문서·텍스트 파싱·요약·생성 | 의뢰·입력 문서 해석, 보고서 자동작성 |
| 비전 VLM | 화면·이미지·도면 판독, GUI 화면 인식 | 스크린샷·도면 판독, 화면 기반 작업 |
| 임베딩 검색 | 의미 기반 유사 문서·사례 검색 | 과거 사례 매칭, 참조값 조회 |
| computer-use 제어 | 화면을 보고 GUI를 직접 클릭·입력 | 사람 대신 시스템을 조작하는 단계 |

> 특정 모델명을 강요하지 않는다. 외부 LLM API가 차단된 환경이면 사내 OpenAI-호환 VLM/LLM/임베딩 엔드포인트로 대체하고, 가용한 모델명을 이 칸에 적는다. 가용 모델이 다르면 To-Be 자동화 범위도 달라진다.

### ③ 운영 기준 정의 — 신뢰도 임계·재시도·승인 게이트 (빈 양식)

> 작성 가이드: 🤖 자동 수행이 폭주하지 않도록 각 단계를 **게이팅**하는 변수를 정의한다. 변수 기호는 #09 KSF 수용 기준과 **동일하게** 쓴다(설계 일관성). "어떻게 정하나"는 아래 가이드를 따른다.

| 변수 | 의미 | 적용 위치 | 어떻게 정하나 (가이드) |
|------|------|-----------|------------------------|
| **θ** | 화면·작업 인식 신뢰도 임계 | 〈AI 인식·제어 단계〉 | AI 출력 신뢰점수가 θ 미만이면 재시도/핸드오프. 처음엔 보수적(높게)으로 두고 운영하며 낮춘다 |
| **τ** | 결과 정확도 임계 | 〈핵심 판정 단계〉 | "이 정확도면 사람 승인 없이 넘겨도 되는" 최소선. #09 수용 기준·PoC 측정 변수와 같은 값 |
| **r** | 재시도 횟수 임계 | 〈AI 단계 공통〉 | r번 재시도해도 θ·τ 못 넘으면 예외 핸드오프. 보통 1~2회로 시작 |
| **승인 게이트** | 사람 최종 승인 없이는 〈산출물 확정〉 불가 | 〈마지막 단계〉 | 초기 운영은 100% 사람 승인, 신뢰가 쌓이면 조건부 자동 승인으로 완화 |

**완전자동화 아님(Human-in-the-loop) 박제**: AI 연속 구간은 〈θ·τ·r〉 안에서만 자동이고, 진입 검토·최종 승인·예외 핸드오프는 항상 사람이 통제권을 갖는다. 이 경계가 #06 적용범위·#02 AI 대체 판정(부분 자동화)·#09 선결조건과 정렬돼야 한다.

## 퍼실리테이션 가이드 (How to run)

### 진행 순서

1. **준비(5분)** — #07 As-Is 플로우차트와 단계 테이블을 벽에 붙인다. 그 위에 🤖/🧑 레인 두 줄을 가로로 긋는다. 같은 단계 축을 재사용하는 것이 핵심.
2. **발산(20분)** — 단계마다 "AI가 들어오면 이게 어떻게 바뀌나?"를 던지고, 각 단계 박스를 🤖 또는 🧑 레인으로 옮긴다. 어떤 AI 기술이 필요한지(생성/비전/검색/제어)를 위 "적용 AI 기술 보기"에서 고른다.
3. **수렴(20분)** — 🤖 구간이 정해지면 **"AI는 어디까지, 사람은 언제 개입하나"** 를 명시한다. 예외가 어디서 사람에게 넘어가는지(핸드오프) 화살표를 그리고, 운영 기준 변수(θ/τ/r/승인게이트)를 채운다.
4. **정리(15분)** — 확장 테이블에 단계별 To-Be 수행·기술·수행주체·s′·시스템·운영 기준을 옮긴다. 레인 전환 경계가 #06 적용범위와 글자 그대로 일치하는지 검산한다.

### 참가자에게 던질 질문

- "AI가 들어오면 **이 단계가 구체적으로 어떻게 바뀌나?** (없어지나, 사람이 검토만 하나, 완전히 AI가 하나)"
- "이 단계에 **어떤 AI 기술이 필요한가?** 텍스트 생성인가, 화면·이미지 판독(비전)인가, 유사사례 검색인가, GUI 직접 제어인가?"
- "**AI는 어디까지 하고, 사람은 언제 개입**하나? 시작점과 끝점은 누가 잡나?"
- "**예외는 언제 사람에게 넘기나?** AI 결과가 어느 신뢰도/정확도(θ·τ) 아래면 사람이 받아야 하나? 몇 번 재시도(r) 후 포기하나?"
- "**최종 확정에 사람 승인이 필요한가(승인 게이트)?** 없이 자동 등록해도 되는 단계는 어디인가?"

### 완료 기준 (Definition of Done)

- 모든 단계가 🤖 또는 🧑 레인에 배치됐고, 둘 중 어느 쪽도 **전부를 독점하지 않는다**(사람 개입점 ≥ 1).
- 🤖 구간 → 🧑 예외 핸드오프 화살표가 1개 이상 그려졌다.
- 운영 기준 변수(θ/τ/r/승인게이트)가 정의되고 #09와 기호를 공유한다.
- 🤖/🧑 레인 전환 경계가 #06 적용범위와 일치한다.

### 타임박스

총 **60분 내외** (준비 5 / 발산 20 / 수렴 20 / 정리 15). 단계가 많으면 발산을 늘리지 말고 🤖/🧑 판정이 모호한 단계만 수렴에서 집중 토론한다.

## 흔한 실수 · 체크포인트

- **완전자동화로 그림** — 모든 단계를 🤖 레인에 넣고 사람을 지운다. → 진입 검토·최종 승인·예외 처리 중 **최소 한 곳은 사람**. (통과 기준: 🧑 레인에 박스 ≥ 1, 핸드오프 화살표 존재)
- **사람 개입점 누락** — AI 구간만 정교하게 그리고 "AI가 틀리면 누가 받나"를 안 그린다. → 예외 핸드오프 경로를 반드시 그린다. (통과 기준: 운영 기준 미달 → 🧑 화살표가 명시됨)
- **운영 기준 공백** — 🤖 구간을 "AI가 알아서"로 두고 θ/τ/r/승인게이트를 비운다. → 자동 수행은 게이팅 변수 없이는 폭주한다. (통과 기준: ③ 운영 기준 표의 모든 변수가 정의됨)
- **적용범위 불일치** — 레인 전환 경계가 #06 과제 정의서의 AI 대상 범위와 어긋난다. → 글자 그대로 맞춘다. (통과 기준: 🤖 구간 = #06 적용범위)
- **소요시간을 지어냄** — s′를 임의 수치로 채워 ΔS를 부풀린다. → s′는 placeholder로 두고 담당자가 확정. (통과 기준: 미확정분이 s′로 표기됨)
