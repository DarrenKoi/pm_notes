# 06 · AI 과제 정의 (Agent Definition) — 템플릿

> 한 줄: Discovery(01~05)의 흩어진 산출물을 **AI 과제 정의서 1매**로 수렴시켜, "무엇을·왜·AI가 어디까지" 하는 과제를 실행 단위로 확정한다.

## 이 단계가 무엇인가 (Why)

앞의 다섯 단계(전략 일관성→대상 업무→Pain Point→근본원인→적정성)는 각각 따로 놓인 산출물이다. Step8은 이것들을 **단 한 장(1매)으로 압축**해, 처음 보는 사람도 "이 과제가 무엇이고, 왜 하며, AI가 어디까지 하는가"를 즉시 읽게 만든다.

- **추적성의 도착지** — 정의서의 각 칸은 앞 단계 산출물이 흘러들어오는 종착점이다. 업무목표는 01, 핵심업무 구조화는 02, 추진배경(Pain)은 03, 근본원인(Big Why)은 04에서 오고, 적정성 점수(05)는 정의서를 써도 되는지 판정하는 **착수 게이트**다. 이 매핑이 끊기면 과제는 "개인 흥미"로 전락한다. (출처: [방법론 틀 02 Step8](../02-기술문서_AI과제정의구현.md), 양식 [원문 09](../../lectures/captures/09-ai-task-definition-template.md))
- **역할 경계의 박제** — AI Agent 칸(대상업무·요구사항·적용범위·기대효과)에서 AI 수행 구간과 사람 구간(예외·승인)을 못 박아, **완전자동화가 아닌 Human-in-the-loop(HITL)** 임을 정의서 차원에서 고정한다. Pain Point·Big Why를 양식 안에 명시적으로 두는 것은 가치 중심·근본원인 중심 원칙을 양식으로 강제하는 장치다. (출처: [원문 09](../../lectures/captures/09-ai-task-definition-template.md))

## 작성할 산출물 (What)

### ① AI 과제 정의서 (1매) — 빈 양식

> 아래 ASCII 박스를 그대로 복제해, `___` 와 `<…>` 를 워크숍 결과로 채운다. 칸별로 "어느 앞 단계에서 받아오는가"를 화살표(←)로 표기해 두었다.

```
┌──────────────────────────────────────────────────────────────────────────┐
│ 과제명 : ____________________________________________________             │
│ 구분   : ☐ Marketing   ☐ R&D   ☐ Production   ☐ Support                   │
│ 담당 조직 : ______________    담당 임원(Champion) : ______________         │
├──────────────────────────────────────────────────────────────────────────┤
│ [상세내역]                                                                 │
│                                                                            │
│  ■ 업무 목표 (부문/팀)            ← 01 전략 일관성                          │
│     · _______________________________________________________             │
│     · _______________________________________________________             │
│                                                                            │
│  ■ 부문/본부 핵심 업무 구조화      ← 02 대상 업무 영역                      │
│     ______ > ______ > ______  (대상 업무 구간 ①~ⓝ)                       │
│       ① ____________  ② ____________  ③ ____________ …                    │
│                                                                            │
│  ■ 추진 배경 (Pain Point)         ← 03 Pain Point 정량화                   │
│     · 최대 병목 = ____________________________________________             │
│     · (빈도·심각성·주요성 정량 점수 τ는 워크숍에서 확정)                   │
│                                                                            │
│  ■ 근본원인 (Big Why)             ← 04 근본원인 ideation                   │
│     · ________________________________________________________            │
│       → (a) ______________________  → (b) ______________________          │
│     · (집중 근본원인 = ______ — KSF·PoC 리스크의 출처)                     │
│                                                                            │
│  ■ AI Agent                                                                │
│     ─ 대상 업무 : ____________________________________________             │
│                  (AI가 대신 수행하는 구간만 명시 / 사람 구간 제외)         │
│     ─ 요구 사항 : ____________________________________________             │
│                  (필요한 AI 능력: 생성/비전/검색/제어 등)                  │
│     ─ 적용 범위 : ____________________________________________             │
│                  (시작 범위 → 점진 확장 / 예외·신규는 HITL 유지)           │
│     ─ 기대 효과 : ____________________________________________             │
│                  (Pain 해소·처리량·일관성 등 정량/정성 효과)               │
└──────────────────────────────────────────────────────────────────────────┘
```

**작성 가이드**
- **과제명** — 한 문장. "〈AI 기술〉 기반 〈대상 업무〉 자동화/지원" 형태로 명사구화한다.
- **구분** — 4분류(Marketing·R&D·Production·Support) 중 1개에 체크. 대상 업무가 어느 가치사슬에 있는지로 결정.
- **담당 임원(Champion)** — 미정이면 `<담당 임원>` 플레이스홀더로 두고 워크숍에서 지정한다. 스폰서 없는 과제는 실행되지 않는다.
- **업무 목표** — 01 연계표의 "업무 목표" 칸을 그대로 가져온다(조직목표에서 연역된 것이어야 함).
- **핵심 업무 구조화** — 02의 L1>L2>L3 위계와 대상 구간(①~ⓝ)을 옮긴다.
- **추진 배경/근본원인** — 03의 최대 병목과 04의 3-Why 결과·집중 근본원인을 옮긴다. **근본원인 칸을 비우지 않는다**(아래 체크포인트 참조).
- **AI Agent 4칸** — 03(병목)+04(근본원인)을 "AI가 무슨 작업을, 어떤 능력으로, 어디까지" 하는 사양으로 변환한다. 사람 구간(승인·예외)은 대상에서 명시적으로 뺀다.

## 퍼실리테이션 가이드 (How to run)

**타임박스 권장: 60~75분** (1과제 기준)

1. **준비 (5분)** — 01~05 산출물(연계표·구조화·Pain 표·3-Why·스코어카드)을 한 벽에 펼친다. 05 적정성 게이트가 통과됐는지 먼저 확인한다(미통과면 정의 진행 보류).
2. **발산 (20분)** — 정의서 박스를 화면에 띄우고 칸별로 앞 단계 결과를 끌어와 채운다. 칸마다 "이건 어느 단계에서 왔나?"를 소리 내어 확인한다.
3. **수렴 (25분)** — AI Agent 4칸(대상·요구·범위·효과)을 좁힌다. 특히 **대상 업무에서 사람 구간을 빼고**, 적용 범위에 "시작 범위 → 점진 확장"을 명시한다.
4. **정리 (10분)** — 추적성 매핑 표를 채워 모든 칸의 출처가 끊기지 않는지 검산한다. Champion·게이트 통과 여부를 체크리스트로 마감한다.

**참가자에게 던질 질문 (이 단계 고유)**
- "이 과제를 한 문장으로 말하면?" (과제명)
- "이 과제의 스폰서(담당 임원)는 누구인가? 누가 책임지나?" (Champion)
- "AI가 정확히 무슨 작업을 대신하나? 사람은 어디서 손을 떼고 어디서 다시 개입하나?" (대상 업무·HITL)
- "그 작업을 하려면 AI에 어떤 능력이 필요한가?(읽기·생성·검색·제어)" (요구 사항)
- "처음부터 전부 적용하나, 아니면 어디서 시작해 어디로 넓히나?" (적용 범위)
- "이걸 하면 어떤 Pain이 얼마나 줄어드나?" (기대 효과)

**완료 정의(DoD)** — ⓐ 정의서 1매 모든 헤더·상세 칸이 채워짐(근본원인 포함), ⓑ AI Agent 대상에서 사람 구간이 분리되어 HITL이 명시됨, ⓒ 추적성 매핑 표의 모든 칸 출처가 앞 단계로 연결됨, ⓓ 적정성 게이트(05) 통과 확인.

## 흔한 실수 · 체크포인트

- **근본원인 칸을 비운다** — 가장 흔한 함정. Pain만 적고 Big Why를 비우면 "AI로 증상만 가린" 과제가 된다. 04의 3-Why 결과(집중 근본원인)를 반드시 옮긴다. → 통과 기준: 근본원인 칸에 "왜→왜→왜"의 종착점이 1개 이상 적혀 있다.
- **적용 범위가 모호하다** — "전사 적용" 같은 무경계 표현은 실행 불가. "어디서 시작 → 어떻게 확장"과 "예외/신규는 사람 유지"를 명시한다. → 통과 기준: 시작 범위와 확장 방향이 각각 1줄 이상.
- **대상 업무에 사람 구간이 섞인다** — AI 대상에 승인·최종검증·예외처리까지 넣으면 완전자동화로 오해된다. 사람 구간을 명시적으로 빼고 HITL을 박는다. → 통과 기준: "AI 구간 / 사람 구간"이 분리 기술됨.
- **추적성이 끊긴다** — 어떤 칸이 앞 단계 산출물 없이 새로 등장하면, 그 과제는 전략에서 연역된 게 아니다. → 통과 기준: 매핑 표의 모든 행에 출처 링크가 살아 있다.
- **게이트를 건너뛴다** — 적정성(05) 미통과인데 정의서를 확정하면 우선순위 검증이 빠진다. → 통과 기준: 05 종합점수가 착수 임계를 넘었음을 확인한 뒤 확정.
