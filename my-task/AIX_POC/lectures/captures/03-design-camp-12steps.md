# 원문 03 — New AI Design Camp 단계별 구성

> Design Camp 각 진행 단계에서 **무엇을 끌어내고 어떻게 연결되는지**를 유념합니다.

## Track A — AI Agent 적용 Idea 도출

| # | 단계 | 영문 | 핵심 활동 |
|---|------|------|-----------|
| 1 | 부문/본부 핵심 업무 정의 | Task Identification | 업무 영역 식별 및 Level 체계화 |
| 2 | AI Agent 대상 영역 분석 | Scope Analysis | AI 개입 최소 단위까지 분해 |
| 3 | AI 구상도 맵핑 (Optional) | Target Scoping | 핵심 KPI 연계 대상 영역 선정 |
| 4 | Pain Point 도출 | Problem Structuring | Pain Point 구조화 |
| 5 | 근본 원인 분석 | Root-Cause Analysis | 3-Whys 기반 근본 원인 파악 |
| 6 | 문제 해결 Ideation | Disruptive Ideation | 혁신적인 아이디어 도출 |
| 7 | 적정성 검토 | Impact Validation | 비즈니스 임팩트 기반 적정성 검증 |

## Track B — AI Agent 실행 Process 구체화

| # | 단계 | 영문 | 핵심 활동 |
|---|------|------|-----------|
| 8 | AI 과제 정의 | Agent Definition | 업무 단위 Agent 역할 정의 |
| 9 | As-Is 업무 프로세스 정의 | Task Breakdown | As-Is 일하는 방식 작성 |
| 10 | To-Be 업무 프로세스 정의 | Biz-Tech Alignment | To-Be 일하는 방식 작성 |
| 11 | Key Success Factor 정의 | Task Prioritization | 실행을 위한 선결 처리 항목 정의 |
| 12 | 개발 일정 수립 | Action Planning | 현실적 일정 수립 및 액션 확정 |
