# 원문 08 — 최종 산출물 예시 (Pain Point → 근본 원인 → 해결 아이디어)

> - 개선 필요성이 높은 Pain Point를 유발하는 **근본 원인**을 분석하고, 이를 해결할 수 있는 방안을 **아이디어 차원**에서 논의합니다.
> - 이 단계는 실행 Process 설계 시 문제 해결 방법을 더 명확하게 설정하도록 합니다.

## ※ AI 대상영역 예시: 마케팅 — 산업/경쟁사 분석

| Pain Point | 근본 원인 | 해결 아이디어 |
|------------|-----------|----------------|
| 데이터 출처별 기준 불일치로 내부 해석 혼선 🔴🔴 | 내부 통합 표준이 없음 / 데이터 관리 주체 부재 및 단일 참조 기준 미정의 | 주요 기관별 단위·정의 Mapping Table 작성 / 단일 참조 기준 제정 + Agent 기반 자동 정합화 |
| 경쟁사 동향·투자 정보 수집의 수작업 의존 🔴🔴 | 소스 분산 but 자동 수집 체계 없음 / 비정형 데이터를 개별 탐색·정리, 정보 수집·정리 프로세스가 개인 업무로 귀속 | 자동 스크랩·이슈 트래킹 시스템 구축 / Market Watcher TF 및 공유 프로세스 설계 / Keyword 기반 Alert |
| 리포트 업데이트 및 최신성 관리 부재 🔴 | 리포트 버전/수집일 등 메타데이터가 없고, 최신 리포트 식별 불가 / 콘텐츠 저장 관리체계 미흡 | 리포트 메타데이터 관리 시스템 구축 (수집일·출처·버전 자동 태깅) / "Latest Only" 필터로 최신판만 자동 표시 |
| 내부 공유 포맷·요약 기준의 일관성 부족 | 담당자마다 요약 스타일·분류기준이 달라 브리핑 품질 편차 발생 / 리포팅 가이드라인·품질관리 체계 부재 | 산업이슈/경쟁사 동향 AI 요약 자동화 / 리포팅 표준 템플릿 및 품질관리 체계 확립 |

🔴 표식 = Pain Point 심각도(고통 정도) 표시로 추정.
